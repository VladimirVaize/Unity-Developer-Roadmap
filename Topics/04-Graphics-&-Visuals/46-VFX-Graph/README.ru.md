# ✨ VFX Graph: Современная система частиц для высокопроизводительных эффектов

VFX Graph (Visual Effect Graph) — это мощный инструмент в Unity, 
основанный на компьютерном шейдере (compute shader) и использующий вычислительные возможности GPU. 
В отличие от классической системы частиц (Shuriken Particle System), 
VFX Graph позволяет создавать сотни тысяч и миллионы частиц с высокой производительностью и сложной логикой поведения.

---

## 🎯 Когда использовать VFX Graph?

| Сценарий | Классическая система | VFX Graph |
| --- | --- | --- |
| Простые эффекты (дым, искры, следы) | ✅ Да | ❌ Избыточно |
| Массовые эффекты (стая птиц, толпа, пыльца) | ❌ Низкая производительность | ✅ Высокая производительность |
| Визуальные спецэффекты (магия, взрывы, симуляция жидкости) | 🟡 Ограничено | ✅ Безгранично |
| Интерактивные эффекты (реакция на игрока, физика) | 🟡 Сложно | ✅ Визуальное программирование |

## 🧠 Ключевые понятия VFX Graph
### 1. Граф (Graph)
Визуальный редактор, где вы создаёте эффекты, соединяя блоки (Blocks) и узлы (Nodes). 
Блоки определяют, что происходит с частицами (движение, цвет, форма), а узлы управляют потоком данных.

### 2. Контексты (Contexts)
Это основные этапы жизни эффекта, расположенные слева направо в графе:
- Initialize — начальные значения частиц (позиция, скорость, цвет, время жизни)
- Update — поведение частиц каждый кадр (силы, гравитация, вращение, столкновения)
- Output — как и где отрисовываются частицы (материал, форма, смешивание)

### 3. Блоки (Blocks)
Перетаскиваются в контексты. Например:
- `Set Position` (Cone) — частицы появляются из конуса
- `Add Velocity` (Random) — случайная начальная скорость
- `Gravity` — притяжение вниз
- `Collide with Scene` — столкновение с геометрией сцены

### 4. Атрибуты частиц
Каждая частица имеет встроенные атрибуты: `position`, `velocity`, `color`, `lifetime`, `age`, `size`, `angle`, `alive` и др. 
Вы можете менять их в блоках.

## ⚙️ Пошаговый пример: Создание огненного взрыва
### 1. Создайте VFX Graph ассет:
`Assets → Create → Visual Effects → Visual Effect Graph`

### 2. Создайте объект в сцене:
`GameObject → Visual Effects → Visual Effect`

### 3. Откройте граф и настройте контексты:
Initialize:
- `Set Position (Sphere)` — радиус 2
- `Set Lifetime (Random)` — от 1 до 2 секунд
- `Set Color (Gradient)` — от жёлтого к красному
- `Set Size (Random)` — от 0.2 до 0.8

Update:
- `Add Velocity (Radial)` — вылетает из центра наружу
- `Gravity` — падает вниз
- `Drag` — замедление

Output:
- Выберите `Quad Output` (плоские спрайты)
- Назначьте материал с текстурой огня и аддитивным смешиванием

### 4. Используйте параметры (Exposed Parameters)
Создайте параметр `SpawnRate` (float) и привяжите к `Rate` в блоке спавна. 
Теперь вы можете менять интенсивность взрыва из инспектора прямо во время игры!

## 🚀 Продвинутые возможности
- Визуальное программирование: создавайте собственную логику с помощью узлов (`Lerp`, `Noise`, `Sample Curve`, `Branch`)
- Взаимодействие со сценой: эффекты реагируют на коллайдеры, триггеры, объекты (через `Block: Collide with Scene` и `Trigger Event`)
- События (Events): запускайте эффект по сигналу из скрипта (`Send Event` на компоненте Visual Effect)
- Блоки чёрного размера (Blackboard): категории параметров, текстур, мешей, градиентов
