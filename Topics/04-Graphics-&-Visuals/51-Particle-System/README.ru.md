# ✨ Система частиц (Particle System) в Unity: Полное руководство для 2D анимации

Система частиц (Particle System) — это мощный компонент Unity для создания динамических 2D/3D эффектов: дыма, огня, искр, дождя, листвы, магии и т.д. 
В 2D-играх она часто используется для фоновой атмосферы, эффектов попаданий, взрывов и взаимодействий.

> 📌 Все параметры находятся в окне Inspector при выборе GameObject с компонентом `Particle System`.

---

## 🧩 1. Модули системы (System Modules)
Модули — это логические блоки, которые можно включать/выключать. 
Каждый модуль добавляет определённое поведение.

| Модуль | Назначение |
| --- | --- |
| Main | Главные настройки: длительность, зацикливание, стартовая скорость, максимальное число частиц. |
| Emission | Управление количеством частиц, испускаемых в секунду (rate) или за один выброс (bursts). |
| Shape | Форма области, из которой рождаются частицы (сфера, конус, прямоугольник, луч и др.). |
| Velocity over Lifetime | Скорость частиц меняется со временем. |
| Color over Lifetime | Цвет частиц плавно изменяется. |
| Collision | Частицы сталкиваются с объектами сцены (твёрдые поверхности, коллайдеры). |
| Sub Emitters | Рождение другой системы частиц при событии (рождении, смерти, столкновении). |
| Renderer | Как рисуется каждая частица (материал, сортировка, освещение). |

---

## 🚀 2. Эмиссия (Emission)
Определяет как и когда появляются частицы.

- Rate over Time — количество частиц в секунду (например, 30 частиц/сек).
- Rate over Distance — частицы на метр движения системы (полезно для колёс, следов).
- Bursts — мгновенный выброс фиксированного числа частиц в определённый момент (взрыв).

📌 *Пример использования*: для непрерывного дыма используйте Rate over Time = 20, а для взрыва — один Burst на 100 частиц при времени 0.

---

## 🔘 3. Форма (Shape)
Определяет геометрию источника частиц.

| Форма | 2D-применение |
| --- | --- |
| Rectangle | Частицы рождаются в прямоугольной области — идеально для дождя или опавших листьев. |
| Circle | Для заклинаний, аур, искр от точечного источника. |
| Edge | Частицы идут по линии — полоса огня, край меча при ударе. |
| Mesh | Сложные формы (редко в 2D). |

> ⚙️ В 2D часто используют `Shape: Rectangle`, ширина и высота задаются в World-единицах.

---

## 💥 4. Коллизии (Collision)
Позволяет частицам взаимодействовать с физическими объектами (стены, платформы, враги).

- Type:
  - `Planes` — бесконечные плоскости (быстро, но грубо).
  - `World` — коллайдеры объектов сцены.
 
- Mode:
  - `Collision` — частица отскакивает или исчезает.
  - `Kill` — частица удаляется при контакте (например, искра об стену).
 
- `Dampen` — потеря скорости после удара.
- `Bounce` — упругость.

🔧 Для 2D-игр важно включить галочку `2D Colliders` в параметрах коллизий, чтобы частицы работали с `BoxCollider2D`, `CircleCollider2D`.

---

## 🌿 5. Подветренная анимация (Sub‑Emitters)
Это возможность создать вложенную систему частиц, которая запускается в ответ на событие в родительской системе.

| Событие | Что происходит |
| --- | --- |
| Birth | При рождении каждой частицы создаётся дочерний эффект. |
| Collision | При столкновении частицы рождается новый эффект (искры, дым). |
| Death | При смерти частицы запускается другой эффект (взрыв, разлетающиеся осколки). |

📌 *Пример*: Частица искры ударяется в стену → по событию Collision создаётся подсистема "дымок", которая быстро затухает.

Для настройки: в модуле `Sub Emitters` нажмите `+`, выберите событие и назначьте другую Particle System (префаб или дочерний объект).

---

## ⚡ 6. Оптимизация частиц (Performance)
Слишком много частиц убивает производительность, особенно на мобильных устройствах.

### ✅ Практические советы:
| Проблема | Решение |
| --- | --- |
| Слишком много живых частиц | Уменьшить `Max Particles` (Main module). |
| Долгоживущие частицы | Использовать модуль `Limit Velocity over Lifetime` и сократить `Start Lifetime`. |
| Сложные коллизии | Для дальних или мелких эффектов отключать `Collision` или использовать простые `Planes`. |
| Много Sub‑Emitters | Избегать цепочек из 3+ уровней, объединять эффекты в один префаб. |
| Рендеринг | Использовать простые материалы (Mobile/Particles‑аддитивные) и маленькие текстуры. |
| Burst-нагрузки | Растягивать выбросы на несколько кадров (уменьшать число частиц в одном Burst). |

> 🔁 Используйте **Particle System Profiler** (окно Profiler → Particles) для поиска узких мест.

## 🧪 7. Пример быстрой настройки (для 2D игры)
Задача: Создать эффект падающих осенних листьев.

1. Создайте `Particle System` → в Main: Loop = true, Start Lifetime = 3 сек, Start Speed = 0.5 (вниз).
2. Emission: Rate over Time = 15.
3. Shape: Rectangle (ширина 10, высота 1) сверху экрана.
4. Color over Lifetime: от жёлтого → красному → прозрачному.
5. Texture Sheet Animation: укажите спрайт листа (лист-атлас).
6. Collision: World + 2D Colliders, Dampen = 0.5 (листья слегка отскакивают от земли).
7. Sub Emitters: при смерти частицы (Death) запускать мелкий эффект "пыльца".
