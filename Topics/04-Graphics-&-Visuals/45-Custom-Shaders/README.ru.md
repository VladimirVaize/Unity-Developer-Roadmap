# 🎨 Кастомные шейдеры в Unity: HLSL / ShaderLab — основы для шейдерных художников и программистов

Этот материал познакомит вас с основами написания кастомных шейдеров в Unity с использованием **ShaderLab** (обёртка) и **HLSL** (язык программирования шейдеров). 
Мы разберём синтаксис, структуру шейдера, а также вершинные и пиксельные (фрагментные) шейдеры.

> Предполагается, что вы уже знакомы с базовым интерфейсом Unity и имеете минимальные навыки программирования (C# не обязателен, но полезен).

---

## 🧱 1. Что такое шейдер и зачем он нужен?
Шейдер — это программа, которая выполняется на видеокарте (GPU) и определяет, как будет выглядеть каждый пиксель или вершина 3D-модели. С помощью шейдеров вы можете создавать:
- эффекты материалов (металл, дерево, пластик);
- искажения, свечение, прозрачность;
- постобработку (размытие, цветокоррекция);
- анимированные текстуры (вода, лава, траву на ветру).
В Unity шейдеры пишутся на **HLSL** (High-Level Shading Language) внутри обёртки **ShaderLab**, которая связывает шейдер с настройками материала в Инспекторе.

---

## 🏗️ 2. Структура шейдера в ShaderLab
Базовый шейдер выглядит так:
```glsl
Shader "Custom/MyFirstShader"
{
    Properties
    {
        _Color ("Main Color", Color) = (1,1,1,1)
        _MainTex ("Texture", 2D) = "white" {}
    }
    
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 200
        
        Pass
        {
            CGPROGRAM  // или HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            
            #include "UnityCG.cginc"
            
            // ... объявление переменных и функций
            
            ENDCG
        }
    }
}
```

### Разбор по частям:

| Часть | Назначение |
| --- | --- |
| `Shader "Имя"` | Имя шейдера, под которым он появляется в меню выбора материала |
| `Properties` | Параметры, видимые в Инспекторе (цвета, текстуры, числа, слайдеры) |
| `SubShader` | Основное тело шейдера. Может быть несколько — для разных возможностей GPU |
| `Pass` | Один проход рендеринга. Можно использовать несколько проходов |
| `CGPROGRAM` / `HLSLPROGRAM` | Блок с кодом на HLSL |
| `#pragma vertex ...` | Указывает функцию вершинного шейдера |
| `#pragma fragment ...` | Указывает функцию пиксельного (фрагментного) шейдера |

> [!Important]
> ⚠️ В современных версиях Unity рекомендуется `HLSLPROGRAM` вместо `CGPROGRAM`.

## 🧬 3. Вершинный шейдер (Vertex Shader)
Вершинный шейдер выполняется для каждой вершины модели. Он отвечает за:
- преобразование координат из локального пространства в экранное;
- передачу данных во фрагментный шейдер (UV-координаты, нормали, цвет);
- простые анимации вершин (например, колебания травы).

### Пример минимального вершинного шейдера:
```hlsl
v2f vert (appdata v)
{
    v2f o;
    o.vertex = UnityObjectToClipPos(v.vertex); // локальные -> экранные
    o.uv = v.uv;                               // передаём UV
    return o;
}
```

Где `appdata` — встроенная структура с данными вершины (позиция, нормаль, UV), а `v2f` — структура, которую вы определяете для передачи во фрагментный шейдер.

## 🎨 4. Пиксельный (фрагментный) шейдер
Фрагментный шейдер выполняется для каждого пикселя (или фрагмента) после растеризации. Он определяет итоговый цвет каждого пикселя на экране.

Пример простейшего фрагментного шейдера:
```hlsl
fixed4 frag (v2f i) : SV_Target
{
    fixed4 col = tex2D(_MainTex, i.uv); // цвет из текстуры
    col *= _Color;                      // умножаем на основной цвет
    return col;
}
```
- `SV_Target` — семантическая привязка к выходному цвету (рендер-таргет).
- `tex2D` — выборка цвета из текстуры по UV-координатам.

---

## 🔁 5. Полный пример простого текстурного шейдера
```hlsl
Shader "Custom/TexturedColor"
{
    Properties
    {
        _Color ("Tint Color", Color) = (1,1,1,1)
        _MainTex ("Albedo (RGB)", 2D) = "white" {}
    }
    
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 200
        
        Pass
        {
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            
            #include "UnityCG.cginc"
            
            // Входные данные вершины
            struct appdata
            {
                float4 vertex : POSITION;
                float2 uv : TEXCOORD0;
            };
            
            // Данные, передаваемые из вершинного во фрагментный шейдер
            struct v2f
            {
                float2 uv : TEXCOORD0;
                float4 vertex : SV_POSITION;
            };
            
            // Свойства из Properties
            sampler2D _MainTex;
            float4 _Color;
            
            v2f vert (appdata v)
            {
                v2f o;
                o.vertex = UnityObjectToClipPos(v.vertex);
                o.uv = v.uv;
                return o;
            }
            
            fixed4 frag (v2f i) : SV_Target
            {
                fixed4 col = tex2D(_MainTex, i.uv);
                col *= _Color;
                return col;
            }
            ENDHLSL
        }
    }
}
```

## 📦 6. Часто используемые функции и встроенные файлы

| Файл / функция | Назначение |
| --- | --- |
| `UnityCG.cginc` | Стандартный набор вспомогательных функций (преобразование координат, освещение) |
| `UnityObjectToClipPos(v)` | Преобразует локальную вершину в позицию на экране (MVP) |
| `tex2D(tex, uv)` | Выборка цвета из текстуры |
| `_Time` | Глобальная переменная времени (для анимаций) |

## 🧪 7. Где применять кастомные шейдеры
- VFX — огонь, вода, телепортация, энергетические щиты.
- Stylized graphics — контурный (toon) шейдер, акварель, пикселизация.
- Анимации на GPU — колышущиеся растения, волны, частицы.
- Постобработка — размытие в движении, эффект старых фильмов.

---

## ⚠️ Важные замечания для новичков
- Шейдеры компилируются под каждую платформу отдельно — тестируйте на целевой (PC / Mobile / Console).
- Избегайте сложных операций во фрагментном шейдере (это может сильно снизить производительность).
- Используйте Shader Graph для визуального создания шейдеров без кода, если вы художник.
- Для отладки используйте `return fixed4(1,0,0,1);` — закрасит всё красным, так легче понять, какой шейдер выполняется.
