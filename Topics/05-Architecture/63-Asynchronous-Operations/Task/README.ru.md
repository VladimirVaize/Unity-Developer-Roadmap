# 🎯 Практическая задача: Асинхронный менеджер загрузки ресурсов
## 📋 Описание задачи
Вам нужно создать систему асинхронной загрузки игровых ресурсов (префабов, сцен, AssetBundle), которая использует все изученные механизмы: 
`AsyncOperation`, `yield return`, обратные вызовы (callbacks), `ResourceRequest` и `AssetBundleRequest`. 
Система должна загружать контент без фризов и уведомлять о прогрессе.

---

## 🧱 Структура задачи
### 📁 Часть 1: Базовый загрузчик с прогрессом
Создайте класс `AsyncLoader` с методами:
- `LoadSceneAsync(string sceneName, Action<float> onProgress, Action onComplete)` — загружает сцену, сообщая о прогрессе (0–1)
- Сцена не должна активироваться, пока прогресс не достигнет 100% (или 0.9, затем allowSceneActivation)

### 📁 Часть 2: Загрузка из Resources
Добавьте метод в `AsyncLoader`:
- `LoadResourceAsync<T>(string path, Action<T> onComplete)` — асинхронно загружает ассет из Resources и вызывает callback с загруженным объектом

### 📁 Часть 3: Загрузка из AssetBundle (симуляция)
Создайте метод:
- `LoadFromAssetBundleAsync(string bundleUrl, string assetName, Action<GameObject> onComplete)`
- Используйте `WWW` или `UnityWebRequest` для загрузки бандла (можно использовать локальный файл для теста: `"file://" + Application.dataPath + "/test_bundle"`)
- Затем асинхронно загрузите ассет через `AssetBundleRequest`

### 📁 Часть 4: Тестовый скрипт с ContextMenu
Создайте скрипт `LoaderTester` с `[ContextMenu]` методами:
- `TestLoadScene` — загружает тестовую сцену (создайте пустую сцену "TestScene")
- `TestLoadPrefab` — загружает любой префаб из Resources (например, куб с компонентом)
- `TestLoadBundle` — загружает ассет из симулированного AssetBundle

### 📁 Часть 5: Обработка нескольких загрузок
Реализуйте очередь загрузок или параллельную загрузку с максимальным лимитом (например, не более 2 одновременных). 
Используйте `yield return` и callbacks для уведомления о завершении всех загрузок.

---

## ✅ Критерии выполнения
1. Все загрузки должны быть асинхронными и не вызывать фризов
2. `LoadSceneAsync` должен правильно использовать `allowSceneActivation` и прогресс
3. `ResourceRequest` используется для загрузки из Resources
4. `AssetBundleRequest` используется для загрузки из AssetBundle
5. Callbacks используются для уведомления о завершении каждой загрузки
6. `[ContextMenu]` позволяет тестировать каждый тип загрузки прямо из редактора
7. (Дополнительно) Отображать прогресс в консоли или на UI-прогресбаре

---

## 🧩 Дополнительное задание (⭐⭐)
Создайте систему кэширования загруженных ассетов:
- `Dictionary<string, object> cache`
- Если ассет уже загружен, при повторном запросе сразу возвращается из кэша (через callback без повторной загрузки)
- Добавьте метод `UnloadAsset(string path)` для выгрузки из кэша

---

## 🧪 Ожидаемый результат
1. Нажатие `TestLoadScene` через ContextMenu → загрузка сцены "TestScene" с выводом прогресса в консоль (0%, 50%, 90%, активация)
2. `TestLoadPrefab` → загрузка и инстанцирование префаба на сцене
3. `TestLoadBundle` → загрузка ассета из бандла (можно создать простой тестовый бандл через AssetBundle Browser)
4. Повторный вызов `TestLoadPrefab` (с кэшем) → мгновенный ответ из кэша
