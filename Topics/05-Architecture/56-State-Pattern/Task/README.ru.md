# 🧪 Практическое задание: Конечный автомат для врага-охранника

Цель: Закрепить навыки реализации паттерна «Состояние» (State Machine) на примере AI врага. 
Вы создадите конечный автомат для охранника с тремя состояниями: `Patrol` (патруль), `Chase` (преследование) и `Attack` (атака).

---

## 📥 Исходные данные
- Проект Unity 2D или 3D (на ваш выбор).
- Простой куб или спрайт для игрока (Player).
- Простой куб или спрайт для врага (Enemy).
- Компонент `Rigidbody2D` / `Rigidbody` для врага (опционально).

---

## 🎯 Задачи
### Этап 1: Создание базовой структуры State Machine
1. Создайте скрипт IState (абстрактный класс или интерфейс) с методами `Enter()`, `Update()`, `Exit()`.
2. Создайте скрипт `StateMachine` с методами `ChangeState()`, `UpdateCurrentState()`.
3. Создайте скрипт `EnemyController`, который содержит ссылки на все состояния и экземпляр `StateMachine`.

### Этап 2: Реализация состояния `Patrol`
Условия:
- Враг движется между двумя точками (или случайным образом по сцене).
- Если расстояние до игрока < 5 единиц → перейти в `ChaseState`.
- Анимация (или просто Debug.Log) «Патрулирую».

### Этап 3: Реализация состояния `Chase`
Условия:
- Враг движется к игроку со скоростью, увеличенной в 1.5 раза.
- Если расстояние до игрока < 1.5 единиц → перейти в `AttackState`.
- Если расстояние до игрока > 8 единиц → вернуться в `PatrolState`.

### Этап 4: Реализация состояния Attack
Условия:
- Враг останавливается и «атакует» (выводит в консоль `Attack!` и наносит 10 урона игроку каждую секунду).
- Если игрок умирает (здоровье ≤ 0) → перейти в `PatrolState` или `IdleState`.
- Если игрок отходит дальше 2 единиц → вернуться в `ChaseState`.

### Этап 5: Система здоровья
1. Добавьте игроку компонент здоровья (например, `PlayerHealth` с полем `int health = 30`).
2. В состоянии `AttackState` при атаке вызывайте `player.TakeDamage(10)`.
3. Реализуйте `DieState` (необязательно), который отключает врага или выводит сообщение.

---

## 🔧 Дополнительные требования (⭐)
1. Анимации: Добавьте простые анимации для каждого состояния (через `Animator` или изменение цвета спрайта).
2. Отладка: Визуализируйте текущее состояние с помощью `TextMeshPro` или изменения цвета врага.
3. UI: Создайте индикатор здоровья игрока (полоска здоровья).

---

## ✅ Критерии успеха
- Враг начинает в состоянии `Patrol` и двигается между точками.
- При приближении игрока враг переходит в `ChaseState` и ускоряется.
- При контакте враг переходит в `AttackState` и наносит урон каждую секунду.
- При отдалении игрока враг возвращается в `PatrolState`.
- (⭐) Анимации меняются в зависимости от состояния.
- (⭐) Индикатор здоровья игрока обновляется при атаке.

---

## 💡 Подсказки
- Для проверки расстояния используйте `Vector3.Distance(transform.position, player.position)`.
- В `ChaseState` можно использовать `MoveTowards` или `Rigidbody.velocity`.
- Не забывайте вызывать `Exit()` для очистки таймеров и корутин при смене состояния.
- Если атака должна быть с задержкой, используйте `InvokeRepeating` или `Coroutine` (но отключайте их в `Exit()`).
