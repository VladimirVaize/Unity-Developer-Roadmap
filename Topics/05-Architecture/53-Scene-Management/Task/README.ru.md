# 🧪 Практическое задание: Система загрузки уровней с глобальным менеджером

Цель: Закрепить навыки работы с LoadScene, аддитивной загрузкой (Additive) и DontDestroyOnLoad. 
Вы создадите игру с главным меню, двумя игровыми уровнями, постоянным UI и глобальным менеджером, сохраняющим счёт.

---

## 📥 Исходные материалы (создаёте сами)
1. Три сцены: `MainMenu`, `Level1`, `Level2`.
2. Простой игровой объект-куб (Player) с движением по стрелкам/WASD.
3. Объект-монета (Coin) на каждом уровне (подбирается при столкновении).
4. Текстовый UI для отображения счёта (Score).

---

## 🎯 Задачи
### Этап 1: Настройка сцен и Build Settings
1. Создайте три сцены в Unity: `MainMenu.unity`, `Level1.unity`, `Level2.unity`.
2. В `MainMenu` добавьте кнопку "Start Game" (через UI Button).
3. Откройте `File` → `Build Settings` → перетащите все три сцены в список (порядок: 0 – MainMenu, 1 – Level1, 2 – Level2).

### Этап 2: Глобальный менеджер (DontDestroyOnLoad)
1. Создайте пустой GameObject → назовите `GameManager`.
2. Добавьте скрипт `GameManager.cs`:
   - Сделайте синглтон (проверка дубликатов).
   - В `Awake()` вызовите `DontDestroyOnLoad(gameObject)`.
   - Храните переменную `int score` и метод `AddScore(int value)`.
  
3. Убедитесь, что `GameManager` не уничтожается при загрузке новых сцен.

### Этап 3: UI-сцена, загружаемая аддитивно
1. Создайте отдельную сцену `UIScene.unity`.
2. В ней создайте Canvas → Text (назовите `ScoreText`).
3. Напишите скрипт `UIManager.cs`, который находит `GameManager` и отображает текущий счёт.
4. В `MainMenu` и `Level1` при запуске загружайте `UIScene` аддитивно:

```csharp
SceneManager.LoadScene("UIScene", LoadSceneMode.Additive);
```

5. Убедитесь, что UI не дублируется при повторной загрузке (проверяйте, загружена ли уже сцена).

### Этап 4: Переход между уровнями с сохранением счёта
1. В `Level1` добавьте финишный триггер (зона, при входе в которую загружается `Level2`).
2. При входе в триггер:
   - Вызовите `SceneManager.LoadScene("Level2")` (заменит Level1 на Level2).
   - Убедитесь, что `GameManager` и `UIScene` НЕ уничтожились (они должны жить благодаря `DontDestroyOnLoad` и аддитивной загрузке).
  
3. В `Level2` добавьте монету (+10 очков) и триггер возврата в `MainMenu`.

### Этап 5: Асинхронная загрузка и прогресс-бар (со звёздочкой ⭐)
1. Замените синхронную загрузку уровней на асинхронную с отображением прогресса.
2. Создайте Canvas с полоской (Slider) в `MainMenu`.
3. При нажатии "Start Game":
   - Покажите слайдер.
   - Запустите корутину `LoadSceneAsync("Level1")`.
   - Обновляйте слайдер из `asyncOperation.progress`.
   - После полной загрузки скройте слайдер.
  
### Этап 6: Выгрузка аддитивной сцены при выходе (дополнительно)
1. При возврате в `MainMenu` из `Level2` выгрузите `UIScene`:
```csharp
SceneManager.UnloadSceneAsync("UIScene");
```

2. Уничтожьте `GameManager` (вызовите `Destroy(GameManager.Instance.gameObject)`), чтобы при новом запуске игры не было старых данных.

---

## ✅ Критерии успеха
- `GameManager` сохраняет счёт между сценами (монеты суммируются).
- UI (счёт) загружается аддитивно и не дублируется.
- Переход `Level1` → `Level2` происходит без потери счёта.
- При возврате в главное меню UI выгружается, GameManager уничтожается.
- (⭐) Асинхронная загрузка работает с прогресс-баром.
