# 🎯 Практическая задача: Система команд для рисования (Drawing App) с Undo/Redo и Реплеем
## 📋 Описание задачи
Вам нужно создать простое приложение для рисования в Unity, 
где игрок может кликать по сетке (grid) и ставить/удалять пиксели. 
Используйте паттерн Команда для реализации:
- Отмены (`Undo`) и повтора (`Redo`) действий
- Записи последовательности действий в файл
- Воспроизведения реплея из файла

---

## 🧱 Структура задачи
### 📁 Часть 1: Базовая команда
Создайте интерфейс `ICommand` с методами `Execute()` и `Undo()`.

Создайте конкретную команду `PlacePixelCommand`, которая хранит:
- `GridManager grid` — ссылку на менеджер сетки
- `int x, int y` — координаты пикселя
- `Color color` — цвет пикселя
- `bool wasOccupied` — был ли пиксель занят до выполнения (для Undo)

### 📁 Часть 2: GridManager
Создайте `GridManager : MonoBehaviour`, который:
- Имеет двумерный массив `Color?[,] grid` (10×10 или 16×16)
- Метод `SetPixel(int x, int y, Color color)` — устанавливает цвет
- Метод `ClearPixel(int x, int y)` — удаляет пиксель
- Метод bool `IsOccupied(int x, int y)` — проверяет, занята ли клетка
- Визуализирует сетку с помощью `OnDrawGizmos` или создаёт спрайты/кубы

### 📁 Часть 3: CommandManager
Создайте `CommandManager`, который:
- Имеет два стека: `undoStack` и `redoStack`
- Метод `ExecuteCommand(ICommand cmd)` — выполняет команду и добавляет в `undoStack`
- Методы `Undo()` и `Redo()`
- При новом `ExecuteCommand()` очищает `redoStack`

### 📁 Часть 4: Рисование через UI
Создайте скрипт `DrawingController`, который:
- Реагирует на клики по клеткам сетки (можно использовать `OnMouseDown` на каждом тайле или Raycast)
- При клике на пустую клетку создаёт `PlacePixelCommand` и выполняет её
- При клике на занятую клетку — удаляет пиксель (ещё одна команда или тот же `PlacePixelCommand` с другим поведением)

### 📁 Часть 5: Сохранение и реплей (⭐ основная сложность)
Добавьте в `DrawingController` или отдельный `ReplayManager`:
- `StartRecording()` — очищает список записанных команд и начинает запись
- `StopRecording()` — останавливает запись, сохраняет команды в файл (JSON)
- `LoadReplay(string filename)` — загружает команды из файла
- `StartReplay()` — сбрасывает сетку в начальное состояние, затем последовательно выполняет все команды из файла с задержками (например, через `Invoke` или `Coroutine`)

### 📁 Часть 6: Горячие клавиши
Добавьте горячие клавиши:
- `Ctrl+Z` — Undo
- `Ctrl+Y` — Redo
- `F1` — начать запись
- `F2` — остановить запись и сохранить
- `F3` — загрузить и запустить реплей

---

## ✅ Критерии выполнения
1. Паттерн Команда реализован корректно: каждая операция рисования/стирания — отдельный объект команды
2. Система Undo/Redo работает последовательно (отмена последних действий и повтор)
3. Реплей воспроизводит все действия в правильном порядке с учётом времени между действиями (или просто последовательно с фиксированной задержкой, например 0.5 секунды)
4. Сохранение и загрузка из файла реализованы через JSON сериализацию

## 🧩 Дополнительное задание (⭐⭐)
1. Добавьте очистку всего поля как одну команду, которую тоже можно отменить
2. Добавьте палитру цветов (3-5 цветов) — команда должна сохранять выбранный цвет
3. Реализуйте мультиплеерный реплей: два игрока рисуют по очереди, реплей показывает оба слоя действий

## 🧪 Ожидаемый результат
После завершения задания:
- При клике на сетку появляются цветные квадраты/точки
- `Ctrl+Z` отменяет последний поставленный или удалённый пиксель
- `Ctrl+Y` возвращает отменённое действие
- При нажатии `F1`, затем нескольких действий, затем `F2` — создаётся файл `replay.json`
- При нажатии `F3` — поле очищается и действия воспроизводятся автоматически (как видео)
- В консоли выводятся логи выполнения команд, Undo, Redo, начала/конца реплея
