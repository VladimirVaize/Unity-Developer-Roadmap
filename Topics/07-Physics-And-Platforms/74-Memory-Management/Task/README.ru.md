# 🎯 Задача: «Оптимизатор памяти текстур для мобильной RPG»
Вы разрабатываете мобильную RPG игру с открытым миром. 
В игре используются сотни текстур различного назначения: UI, персонажи, окружение, эффекты. 
Многие тестировщики жалуются на вылеты на устройствах с 2ГБ ОЗУ. 
Ваша задача — создать систему управления текстурной памятью.

## 📝 Исходные данные:
В игре есть следующие категории текстур:

| Категория | Кол-во текстур | Исходный размер | Текущий формат | Mipmaps |
| --- | --- | --- | --- | --- |
| UI иконки | 50 | 256x256 | RGBA32 | Нет |
| UI фоны | 10 | 2048x2048 | RGBA32 | Нет |
| Персонажи | 20 | 2048x2048 | RGBA32 | Да |
| Окружение | 100 | 1024x1024 | RGBA32 | Да |
| Эффекты | 30 | 512x512 | RGBA32 | Нет |

Текущее потребление памяти (приблизительно):
- UI иконки: 50 × (256×256×32/8) = 50 × 262KB ≈ 13 МБ
- UI фоны: 10 × (2048×2048×32/8) = 10 × 16 МБ = 160 МБ
- Персонажи: 20 × (2048×2048×32/8 × 1.33 с mipmaps) = 20 × 21.3 МБ ≈ 426 МБ
- Окружение: 100 × (1024×1024×32/8 × 1.33) = 100 × 5.33 МБ ≈ 533 МБ
- Эффекты: 30 × 512×512×32/8 = 30 × 1 МБ = 30 МБ

ИТОГО: ~1162 МБ (только текстуры!)

## 📋 Задачи для оптимизации:
### 1. Примените сжатие текстур (самое важное)
Для каждой категории выберите оптимальный формат сжатия:

| Категория | Формат | Почему |
| --- | --- | --- |
| UI иконки | ASTC 4x4 | Максимальное качество для мелких деталей |
| UI фоны | ASTC 6x6 | Хорошее качество для больших областей |
| Персонажи | ASTC 5x5 | Баланс качества и экономии |
| Окружение | ASTC 8x8 | Экономия памяти для фоновых объектов |
| Эффекты | ASTC 6x6 | Хорошо для частиц |

Рассчитайте новую память после сжатия (напишите формулу)

### 2. Оптимизируйте разрешения текстур
Некоторые текстуры имеют избыточное разрешение. Уменьшите:
- UI фоны: с 2048x2048 до 1024x1024 (нет смысла больше экрана)
- Окружение: дальние объекты можно уменьшить до 512x512
- Персонажи: для неигровых NPC — до 1024x1024

### 3. Создайте спрайт-атласы
Объедините в атласы:
- Все UI иконки в один атлас 2048x2048 (ASTC 4x4)
- UI фоны в отдельные атласы (по одному фону на атлас 1024x1024)
- Эффекты частиц в атлас 2048x2048

### 4. Реализуйте систему LOD для текстур (программно)
Напишите код, который:
- Определяет объём ОЗУ устройства
- При < 2ГБ ОЗУ загружает текстуры в половинном разрешении
- При > 3ГБ ОЗУ загружает в полном качестве
- При переполнении памяти выгружает текстуры дальних объектов

### 5. Настройте mipmaps правильно
Исправьте настройки mipmaps для каждой категории:
- UI: Выключить mipmaps (экономия 33% памяти)
- Персонажи: Включить (они двигаются)
- Окружение: Включить (разные расстояния)
- Эффекты: Выключить (обычно близко к камере)

### 6. Напишите мониторинг памяти
Создайте скрипт, который каждые 30 секунд логирует:
- Общую память текстур
- Память по категориям
- Предупреждение при превышении 300 MB

## 💡 Подсказки:
- Для расчёта памяти используйте формулу: `width * height * bitsPerPixel / 8 / 1048576` (в МБ)
- Для сжатия ASTC коэффициент сжатия: для 6x6 — 3.56 бит/пиксель
- В Unity Editor можно эмулировать память через `UnityEditor.EditorPrefs`
- Используйте `Resources.UnloadUnusedAssets()` для принудительной выгрузки
