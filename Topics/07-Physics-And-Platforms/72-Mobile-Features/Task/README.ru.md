# 🎯 Задача: «Мобильный менеджер с жестами и уведомлениями»
Вы разрабатываете мобильную игру-головоломку. Вам нужно реализовать систему, которая обрабатывает касания, отправляет напоминания игроку и экономит батарею.

## 📝 Шаблон для реализации (GameManager.cs):
```csharp
using UnityEngine;
using UnityEngine.UI;

public class MobileGameManager : MonoBehaviour
{
    [Header("UI Elements")]
    public Text gestureDisplayText;
    public Text notificationStatusText;
    
    [Header("Settings")]
    public bool enablePowerSaving = true;
    public int reminderDelaySeconds = 60; // Напоминание через минуту
    
    // ========== ЧАСТЬ 1: Обработка касаний и жестов ==========
    // TODO: Реализовать детектор следующих жестов:
    // 1. Tap (одинарное касание) - вывести "Tap detected!"
    // 2. Double Tap (двойное касание за 300 мс) - вывести "Double tap!"
    // 3. Long Press (касание дольше 0.7 сек) - вывести "Long press!"
    // 4. Swipe вверх/вниз/влево/вправо - вывести направление
    
    // ========== ЧАСТЬ 2: Пуш-уведомления ==========
    // TODO: Реализовать отправку напоминаний:
    // 1. При старте игры запланировать уведомление через reminderDelaySeconds секунд
    // 2. При получении уведомления (запуске из него) обновить notificationStatusText
    // 3. Добавить кнопку для отмены всех уведомлений
    
    // ========== ЧАСТЬ 3: Оптимизация батареи ==========
    // TODO: Реализовать энергосбережение:
    // 1. При сворачивании игры снизить targetFrameRate до 15
    // 2. При разворачивании вернуть на 60
    // 3. Использовать корутину для обновления UI (не каждый кадр)
    // 4. Если enablePowerSaving == true, отключать ненужные компоненты в фоне
}
```

## 📋 Конкретные требования:
1. Детектор жестов (реализовать в отдельном скрипте `GestureDetector.cs`):
   - Отслеживать фазы касания (`Began`, `Moved`, `Ended`, `Canceled`)
   - Вычислять время удержания для определения длительного нажатия
   - Вычислять скорость движения для определения свайпов
   - Отслеживать интервал между касаниями для двойного тапа
  
2. Пуш-уведомления (использовать Unity Mobile Notifications):
   - На Android: создать канал уведомлений с высоким приоритетом
   - На iOS: запросить разрешение
   - При запуске из уведомления загружать особый уровень или отображать сообщение
  
3. Оптимизация:
   - Обновлять UI (отображение жестов) только 10 раз в секунду
   - При потере фокуса приостанавливать все корутины, кроме жизненно важных
   - Использовать `System.GC.Collect()` только при переходе между сценами (не в Update)  

## 🧰 Требования к реализации:
- Создайте три скрипта: `GestureDetector`, `NotificationManager`, `BatterySaver`
- Используйте платформенные директивы для Android/iOS в уведомлениях
- Добавьте комментарии к каждому важному блоку кода
- Все настройки (время удержания, интервал обновления) должны выноситься в поля Inspector

## 🔍 Проверка:
1. Запустите на устройстве — проверьте все жесты
2. Сверните игру — FPS должен упасть
3. Дождитесь напоминания (или ускорьте через инспектор) — должно прийти уведомление
4. Нажмите на уведомление — игра должна открыться и показать сообщение

## 💡 Подсказки:
- Для определения двойного тапа используйте `Time.time - _lastTapTime < 0.3f`
- Для длительного нажатия засекайте время в `Began` и проверяйте в `Ended`
- Для свайпов используйте `touch.deltaPosition` и `touch.deltaTime`
- Не забывайте проверять `Input.touchCount > 0` перед доступом к `Input.GetTouch(0)`
