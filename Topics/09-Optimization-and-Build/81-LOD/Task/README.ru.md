# 🎯 Задача: «Оптимизация города с LOD»
Вы разрабатываете городскую сцену с большим количеством зданий. Вам нужно реализовать систему LOD для 4 типов зданий, которая будет:
1. Автоматически переключать модели в зависимости от расстояния до игровой камеры
2. Плавно переходить между уровнями детализации
3. Адаптироваться под производительность (Dynamic LOD)
4. Использовать биллборды для самых дальних зданий

## 📝 Что нужно реализовать:
### Часть 1: Создание LOD Group для здания
1. Создайте префаб здания с именем `Building_LOD_Template`:
   - `LOD0`: детальная модель (10,000 полигонов, текстуры 4K)
   - `LOD1`: средняя модель (5,000 полигонов, текстуры 2K)
   - `LOD2`: простая модель (1,000 полигонов, текстуры 512x512)
   - `LOD3`: биллборд (плоский спрайт с текстурой здания)
  
2. Настройте компонент LOD Group:
   - LOD0: 70% (0-15 метров)
   - LOD1: 40% (15-40 метров)
   - LOD2: 15% (40-80 метров)
   - LOD3: 5% (80-120 метров)
   - Culled: <5% (>120 метров)
  
### Часть 2: Скрипты управления LOD
3. Напишите скрипт `DynamicCityLOD.cs`:
   - При низком FPS (<30) автоматически снижайте пороги LOD на 20%
   - При высоком FPS (>55) повышайте пороги на 15%
   - Логируйте изменения в консоль (только в Development Build)
  
4. Напишите скрипт `BuildingLODController.cs`:
   - Добавьте метод `SetForcedLOD(int level)` для принудительного переключения
   - При выходе из игры сохраните последний активный LOD
   - Реализуйте событие `OnLODChanged`
  
### Часть 3: Адаптивный LOD
5. Реализуйте систему "важных" зданий:
   - Здания с тегом `ImportantBuilding` должны иметь повышенный LOD (сдвиг на один уровень вверх)
   - Например: LOD2 становится LOD1, LOD1 становится LOD0
  
6. Добавьте эффект "фокуса" камеры:
   - Если здание находится в центре экрана более 2 секунд — временно повысить LOD
  
### Часть 4: Визуальная обратная связь
7. Создайте отладочный режим (клавиша `F1`):
   - Раскрашивайте здания в зависимости от активного LOD:
     - LOD0: Зеленый
     - LOD1: Желтый
     - LOD2: Оранжевый
     - LOD3: Красный
     - Culled: Черный
    
8. Выводите на экран (GUI) информацию:
   - Всего зданий
   - Количество зданий на каждом LOD уровне
   - Текущий FPS
   - LOD Bias
  
---

## 🧰 Требования к реализации:
- Используйте минимум 3 LOD уровня + биллборд
- Обязательно используйте Cross Fade для плавных переходов
- Реализуйте Dynamic LOD на основе FPS
- Добавьте Impostor (биллборд) для самого дальнего уровня
- Используйте `RecalculateBounds()` для корректной работы occlusion culling

---

## 🔍 Проверка работоспособности:
1. Проверьте, что модели переключаются при движении камеры
2. Проверьте, что нет мерцания при переключении (Cross Fade работает)
3. Искусственно снизьте FPS (добавив нагрузку) — убедитесь, что LOD пороги меняются
4. Отметьте здание как ImportantBuilding — должно отображаться детальнее
5. В отладочном режиме цвета должны соответствовать активному LOD
