# 🎯 Задача: «Оптимизация подземелья через Occlusion Culling»
Вы разрабатываете 3D-игру в жанре "подземелье-квест". Сцена состоит из 50+ комнат, соединённых коридорами. 
Без оптимизации FPS падает до 15-20. Ваша задача — внедрить Occlusion Culling для повышения производительности.

### 📝 Исходные данные:
- Сцена содержит: стены (окклюдеры), сундуки, врагов, декоративные объекты
- Камера — от первого лица, движется по подземелью
- Игрок может открывать двери и активировать порталы

---

## 📋 Часть 1: Подготовка сцены (5 шагов)

### 1. Отметьте статические объекты:
- Все стены, полы, потолки, колонны — включите флаг `Static`
- Убедитесь, что они имеют `MeshRenderer` с включёнными тенями (`Cast Shadows = On`)

### 2. Динамические объекты:
- Сундуки, враги, лампы — оставьте НЕ статическими (флаг `Static` выключен)

### 3. Создайте Occlusion Areas:
- Для каждой большой комнаты добавьте `Occlusion Area`
- Настройте размер области под размер комнаты

### 4. Добавьте Occlusion Portals:
- На каждый дверной проём добавьте компонент `Occlusion Portal`
- Настройте поворот так, чтобы портал "смотрел" в проём

### 5. Проверьте теги:
- Окклюдерам (стены) назначьте тег `Occluder`
- Окклюди (декор) — тег `Occludee`

---

## 📋 Часть 2: Настройка бейкинга
6. Откройте окно Occlusion Culling (Window → Rendering → Occlusion Culling)
7. Установите параметры бейкинга:
```text
Smallest Occluder = 1.5
Smallest Hole = 0.5
Backface Threshold = 100
```
8. Нажмите Bake
9. После бейкинга убедитесь, что в папке `Assets/` появился файл `OcclusionCullingData.asset`

---

## 📋 Часть 3: Скриптовое управление (написать код)
Вам нужно реализовать следующие скрипты:

### 🔧 Скрипт 1: `OcclusionDebugger.cs`
Выводит в консоль каждые 2 секунды статистику:
```text
[Occlusion] Visible objects: 45 / 320 (14% culled)
[Occlusion] Current FPS: 58.2
```

Требования:
- Использовать `Renderer.isVisible` для подсчёта
- Использовать платформенную директиву `#if UNITY_EDITOR` для вывода в редакторе

### 🚪 Скрипт 2: `DynamicPortalController.cs`
Управляет Occlusion Portal на двери:
- При открытии двери — портал открывается

### 🏆 Скрипт 3: `RoomOcclusionTrigger.cs`
Срабатывает при входе в комнату:
- Включает динамические объекты в этой комнате
- Выключает динамические объекты в соседних комнатах (на расстоянии > 20 метров)

Алгоритм:
1. При входе в триггер комнаты, получить список всех `GameObject` с тегом `RoomObject` в этой комнате
2. Для каждого объекта: `SetActive(true)`
3. Для объектов в других комнатах: `SetActive(false)` (кроме тех, где есть игрок)

## 📋 Часть 4: Визуализация и тестирование
10. В окне Occlusion Culling → Visualization включите Visualize → Occlusion Culling
11. Выберите камеру игрока в поле Camera
12. Проверьте:
    - В коридоре — видны красные объекты (скрыты) за стенами?
    - В комнате с врагом — враг должен быть зелёным (виден)
    - При выходе из комнаты — враг должен стать красным
   
13. Замерьте FPS до и после:
```text
Без OC: 18 FPS
С OC:   52 FPS
```
