# 🎯 Задача: «Оптимизация леса с 10,000 деревьями»
Вы разрабатываете открытый мир RPG. В сцене нужно разместить 10,000 деревьев, 
но производительность падает до 5-10 FPS. Ваша задача — оптимизировать рендеринг 
с помощью Static Batching и GPU Instancing.

## 📝 Что нужно реализовать:
### Часть 1: Создание тестовой сцены
1. Создайте простой меш дерева (Cylinder + Sphere или импортируйте модель)
2. Создайте материал для дерева (включите GPU Instancing)
3. Создайте скрипт `TreeSpawner.cs`

### Часть 2: Три режима рендеринга
Реализуйте возможность переключаться между тремя режимами:

| Режим | Описание |
| --- | --- |
| Standard | Обычное создание 10,000 GameObject с компонентом MeshRenderer |
| Static Batching | Те же объекты, но помеченные как Static и объединённые |
| GPU Instancing | Использование `Graphics.DrawMeshInstanced` без GameObject |

### Часть 3: Функциональность
Скрипт `TreeSpawner` должен:

1. Генерировать деревья в случайных позициях на плоскости (размер 500x500)
2. Переключать режимы с помощью клавиш `1`, `2`, `3`
3. Показывать статистику на UI:
   - FPS
   - Draw Calls (через `UnityStats.drawCalls`)
   - Количество деревьев
   - Текущий режим
  
4. Анимация деревьев (только для GPU Instancing):
   - Каждое дерево слегка покачивается от ветра (через шейдер)
   - Используйте `MaterialPropertyBlock` для передачи фазы анимации
  
### Часть 4: Оптимизированный шейдер
Создайте шейдер для GPU Instancing, который поддерживает:
- Разный цвет (зелёный → жёлтый → коричневый) для разнообразия
- Анимацию покачивания от ветра (используйте `_WindStrength` и `_Time`)

### Часть 5: Анализ производительности
Запустите каждый режим и запишите:
- Средний FPS
- Количество Draw Calls
- Потребление памяти (RAM)
- Сделайте выводы в консоли при переключении режимов.

---

## 🧰 Требования к реализации:
- Используйте один материал для всех деревьев
- В режиме GPU Instancing используйте `MaterialPropertyBlock` для задания цвета и фазы анимации
- Статические объекты должны быть помечены как `isStatic = true`
- Для статического батчинга вызовите `StaticBatching.Combine()`
- Добавьте UI-кнопки для переключения режимов

---

## 🔍 Проверка работоспособности:
1. Режим Standard — 10,000 отдельных объектов → FPS < 15
2. Режим Static Batching — 1 Draw Call → FPS ~ 60
3. Режим GPU Instancing — ~10 Draw Calls (10,000 / 1023) → FPS ~ 50-60
4. Визуально деревья должны выглядеть одинаково во всех режимах
5. Анимация ветра работает только в GPU Instancing

## 🏆 Бонусное задание:
Добавьте LOD (Level of Detail) для GPU Instancing:
- Деревья ближе к камере — полная детализация
- Дальние деревья — упрощённый меш (например, крест из 2 плоскостей)
- Используйте `Graphics.DrawMeshInstanced` с разными мешами для разных дистанций
