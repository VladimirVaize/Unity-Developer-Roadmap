# 🎯 Задача: «Сетевая игра "Кликер-соревнование" (Clicker Race)»
Вы разрабатываете простую многопользовательскую игру, где игроки соревнуются в скорости кликов. Побеждает игрок, который первым наберёт 10 очков.

## 📝 Что нужно реализовать:
### Часть 1: Базовая настройка сети
1. Создайте новый 3D проект в Unity (версия 2022.3 или новее)
2. Установите пакет `com.unity.netcode.gameobjects` через Package Manager
3. Создайте папки `Scripts/` и `Prefabs/` в `Assets`
4. Создайте `NetworkManager` GameObject и настройте его:
   - Добавьте компонент `NetworkManager`
   - Выберите `UnityTransport` как протокол
   - Создайте префаб игрока (Capsule или Cube) с `NetworkObject`
   - Назначьте префаб в поле `PlayerPrefab`
  
### Часть 2: Серверная логика игры (GameManager)
5. Создайте скрипт `GameManager` на основе `NetworkBehaviour`:
   - `NetworkVariable<int> targetScore = 10` — цель для победы
   - `NetworkVariable<int> gameState` — 0 (ожидание), 1 (игра), 2 (финиш)
   - Метод `StartGameServerRpc()` — запускает игру (только сервер)
   - Метод `EndGameServerRpc(ulong winnerId)` — завершает игру (только сервер)
  
6. Добавьте логику автоматического запуска:
   - При подключении второго игрока игра начинается автоматически
   - Используйте `NetworkManager.ConnectedClientsIds` для подсчёта игроков
  
### Часть 3: Клиентская логика игрока (PlayerController)
7. Создайте скрипт `PlayerController` на основе `NetworkBehaviour`:
   - `NetworkVariable<int> score = 0` — счёт игрока
   - Метод `ClickServerRpc()` — отправляет запрос на клик серверу
   - Валидация на сервере: клик засчитывается только если игра активна
  
8. Реализуйте проверку на победу:
   - При достижении `score >= targetScore` отправляется RPC на сервер
   - Сервер объявляет победителя и завершает игру
  
### Часть 4: Управление вводом
9. На клиенте обрабатывайте:
    - Нажатие левой кнопки мыши (или пробела) → `ClickServerRpc()`
    - Только если игрок является владельцем (`IsOwner`)
  
10. Отключайте ввод для не-владельцев:
```csharp
public override void OnNetworkSpawn()
{
    enabled = IsOwner; // Отключаем скрипт для не-владельцев
}
```

### Часть 5: UI и отображение состояния
11. Создайте Canvas с элементами:
    - Текст статуса игры ("Ожидание игроков", "Игра идёт", "Победил: Имя")
    - Текст счёта для каждого игрока
    - Кнопка "Начать" (активна только для хоста)
   
12. Используйте `NetworkVariable` для синхронизации состояния игры:
    - Обновляйте UI через `OnNetworkSpawn()` и события изменения переменных
   
### Часть 6: События и уведомления
13. Реализуйте RPC для уведомлений:
    - `NotifyPlayerJoinedClientRpc(string playerName)` — новый игрок подключился
    - `NotifyScoreUpdateClientRpc(ulong playerId, int newScore)` — обновление счёта
    - `NotifyWinnerClientRpc(ulong winnerId)` — победитель объявлен
   
14. Используйте `[Rpc(SendTo.NotServer)]` для отправки уведомлений всем клиентам

### Часть 7: Тестирование
15. Протестируйте игру в редакторе:
    - Запустите Host (сервер + клиент) → создайте 2 окна Game View
    - Подключите второй клиент (через кнопку Client)
    - Проверьте, что оба игрока видят друг друга
    - Проверьте механику кликов и подсчёт очков
    - Убедитесь, что сервер валидирует все действия
   
---

## 🧰 Требования к реализации:
- Используйте ServerRpc для всех действий игрока
- Используйте ClientRpc для уведомлений и обновлений
- Используйте NetworkVariable для состояния игры и счёта
- Сервер должен быть авторитетным — все решения принимаются на сервере
- Учтите: `IsOwner` определяет, кому принадлежит объект
- В `DEVELOPMENT_BUILD` выводите логи всех сетевых событий

---

## 🔍 Проверка работоспособности:
1. Запустите игру как Host — должен появиться игрок
2. Запустите второй экземпляр как Client — подключается к серверу
3. При подключении второго игрока игра автоматически начинается
4. Клик мышкой увеличивает счёт на 1
5. При достижении 10 очков игра завершается и объявляется победитель
6. После окончания игры клики не засчитываются
