# 🎯 Задача: «GPU-ускоренная симуляция звёздного скопления»
Вы разрабатываете космический симулятор "StarCluster", в котором нужно симулировать гравитационное взаимодействие тысяч звёзд в реальном времени. 
CPU не справляется с N-Body симуляцией (> 10,000 объектов). Используйте Compute Shaders для расчёта гравитации, движения и рендеринга звёзд.

## 📝 Что нужно реализовать:
### Часть 1: Структуры и буферы
1. Создайте структуру `Star` в Compute Shader:
```hlsl
struct Star
{
    float3 position;
    float3 velocity;
    float mass;
    float luminosity;
    float temperature;
    uint color;  // Packed color (RGBA)
};
```
2. Создайте два буфера в C#:
   - `starBuffer` — RWStructuredBuffer для чтения/записи
   - `colorBuffer` — выходной буфер для цветов (или рендеринг напрямую)
  
### Часть 2: Симуляция гравитации
3. Реализуйте kernel `ApplyGravity`:
   - На каждой итерации вычисляйте силу гравитации от всех звёзд к каждой звезде (N-Body)
   - Используйте смягчение (softening): `dist = max(dist, 0.01f)` чтобы избежать сингулярности
   - Обновляйте скорость и позицию
  
4. Реализуйте kernel `ResolveCollisions`:
   - Проверяйте столкновения между звёздами
   - При столкновении объединяйте звёзды (слияние) или отталкивайте
   - Используйте коэффициент восстановления `0.3f`
  
### Часть 3: Рендеринг
5. Реализуйте рендеринг звёзд через GPU Instancing:
   - Используйте `Graphics.DrawMeshInstancedProcedural`
   - Передавайте позиции и цвета через буферы
   - Размер звезды зависит от массы
  
6. Создайте шейдер `StarShader.shader`:
   - Принимает позиции и цвета из буфера
   - Рисует звезду как точку с освещением (glow)
   - Цвет зависит от температуры (от красного до синего)
  
### Часть 4: Управление и взаимодействие
7. Реализуйте управление камерой:
   - Вращение вокруг центра скопления
   - Приближение/отдаление
   - Добавьте звёзды по клику (новые объекты в буфере)
  
8. Добавьте две модели симуляции:
   - `Spiral Galaxy` — звёзды распределены по спирали
   - `Globular Cluster` — сферическое скопление
  
### Часть 5: Оптимизация
9. Оптимизируйте N-Body симуляцию:
    - Используйте Barnes-Hut или Particle Mesh для ускорения
    - Или используйте приближение: разбить на ячейки и вычислять силы от групп
  
10. Реализуйте Level of Detail (LOD):
    - Близкие звёзды — детальный рендеринг
    - Далёкие звёзды — рендеринг как точки
   
### Часть 6: Профилирование и отладка
11. Выведите в UI:
    - Количество звёзд
    - FPS
    - Время выполнения Compute Shader на кадр
   
12. Добавьте возможность экспорта данных:
    - Сохранять позиции звёзд в JSON или CSV
    - Загружать сохранённую конфигурацию
   
---

## 🧰 Требования к реализации:
- Минимальное количество звёзд: 10,000
- Максимальное количество звёзд: 100,000+
- Frame Rate: 60 FPS на современных GPU
- Используйте ComputeBuffer с флагом `ComputeBufferType.Structured`
- Для случайной инициализации используйте GPU Random (не на CPU)
- Реализуйте переключение между симуляциями без перезагрузки

---

## 🔍 Проверка работоспособности:
1. Запустите симуляцию с 10,000 звёзд
2. Проверьте, что FPS > 30 на среднем GPU
3. Проверьте, что звёзды движутся плавно и взаимодействуют
4. Проверьте, что при клике добавляется новая звезда
5. Проверьте, что переключение моделей работает корректно
6. Запустите Unity Profiler и проверьте время GPU
