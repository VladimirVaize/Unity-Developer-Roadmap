# 🎯 UnityEvents: Настройка событий в инспекторе для слабой связанности (loose coupling)

## 📘 Что такое UnityEvent?
UnityEvent — это специальный тип переменной в Unity, который позволяет привязывать вызовы методов разных компонентов прямо в окне Инспектора (Inspector). 
Это мощный инструмент для создания слабой связанности (loose coupling) между объектами и скриптами.

Слабая связанность означает, что один скрипт не зависит напрямую от другого через ссылки (`GetComponent`, `public` поля на конкретные типы). 
Вместо этого он просто "объявляет событие", а кто и как на него реагирует — настраивается в редакторе визуально.

---

## 🧠 Зачем это нужно?
- ✅ Разделение ответственности: Скрипт, который генерирует событие (например, нажата кнопка, игрок коснулся триггера), не должен знать, какие именно действия последуют.
- ✅ Гибкость: Дизайнер или программист может менять реакцию на событие без изменения кода — просто перетаскивая объекты в инспекторе.
- ✅ Переиспользование: Один и тот же скрипт может использоваться в разных сценах и проектах с разными "слушателями".
- ✅ Визуальная отладка и настройка: Всё настраивается через UI редактора, что удобно для не-программистов.

## 🔧 Как использовать UnityEvent (пошагово)

### 1. Объявите событие в вашем скрипте
```csharp
using UnityEngine;
using UnityEngine.Events;

public class ButtonPresser : MonoBehaviour
{
    // Объявляем публичное событие — оно появится в инспекторе
    public UnityEvent OnButtonPressed;

    void Update()
    {
        if (Input.GetKeyDown(KeyCode.Space))
        {
            // Вызываем событие. Все привязанные методы выполнятся автоматически
            OnButtonPressed.Invoke();
        }
    }
}
```

### 2. Настройте событие в Инспекторе Unity
- Прикрепите скрипт к любому GameObject (например, пустому объекту `EventSystem` или `Manager`).
- В окне Инспектора вы увидите поле `On Button Pressed` (или имя вашего события).
- Нажмите «+» внизу списка событий.
- Перетащите в пустое поле (целевой объект) любой GameObject на сцене, у которого есть нужный метод.
- Выберите из выпадающего списка компонент, а затем конкретный метод (например, `Light.enabled = true` или `Animator.Play("Jump")`).

### 3. Поддерживаемые типы методов
UnityEvent может вызывать:
- Методы без параметров (`void MyMethod()`)
- Методы с одним параметром (`void MyMethod(bool arg)`, `void MyMethod(int arg)`, `void MyMethod(string arg)`, `void MyMethod(float arg)` и др.)
- Статические методы
- Методы, унаследованные от `MonoBehaviour`

## 💡 Примеры использования

### Пример 1: Дверь, открывающаяся по триггеру
#### Скрипт `DoorTrigger.cs`:
```csharp
public class DoorTrigger : MonoBehaviour
{
    public UnityEvent OnPlayerEnter;
    
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
            OnPlayerEnter.Invoke();
    }
}
```
#### Настройка в инспекторе:
- Привязать `OnPlayerEnter` к анимации двери → `Animator.Play("Open")`
- Также добавить второй метод → `AudioSource.Play()` для звука открытия

### Пример 2: Кнопка UI, управляющая несколькими действиями
#### Скрипт `UIManager.cs`:
```csharp
public class UIManager : MonoBehaviour
{
    public UnityEvent OnRestartButtonClicked;
    
    public void RestartGame() // публичный метод, вызываемый из UI Button
    {
        OnRestartButtonClicked.Invoke();
    }
}
```

#### Настройка:
- Привязать `OnRestartButtonClicked` к перезагрузке сцены (`SceneManager.LoadScene`)
- И к сбросу счёта (`ScoreCounter.ResetScore()`)
- И к воспроизведению звука (`Audio.PlayOneShot`)

Без единой строки кода в `RestartGame`, кроме вызова события!

## 🆚 UnityEvent против прямых ссылок

| Прямые ссылки (жёсткая связь) | UnityEvent (слабая связь) |
|----------------------------|----------------------------------|
| `public Light myLight;`<br>`myLight.enabled = true;` | `public UnityEvent OnReachedGoal;`<br>`... OnReachedGoal.Invoke();` |
| Скрипт знает точный тип (`Light`) | Скрипт ничего не знает о слушателях |
| Чтобы изменить действие — нужно менять код | Чтобы изменить действие — перетаскиваешь объекты в инспекторе |
| Нельзя вызвать несколько методов без кода | Можно привязать 10 методов без кода |
| Сложно переиспользовать | Легко переиспользовать |

## ⚠️ Важные замечания
- UnityEvent не сохраняет привязки между сессиями — если удалить объект-слушатель, ссылка в инспекторе станет пустой (но не вызовет ошибку в рантайме, просто ничего не произойдёт).
- Производительность — для событий, происходящих очень часто (каждый кадр), лучше использовать стандартные C# события (public event Action ...).
  UnityEvent немного медленнее, но для UI и игровых событий (открытие двери, смерть игрока) это незаметно.
- Сериализация — UnityEvent полностью поддерживает сохранение в префабах и сценах.

## 🎓 Итог
UnityEvent — идеальный инструмент, когда:
- Вы хотите, чтобы дизайнер уровня мог настраивать поведение без доступа к коду.
- У вас есть один источник события (например, кнопка), но много возможных реакций.
- Вы строите систему, которую легко расширять, не трогая существующие классы.

Используйте UnityEvents для слабой связанности, и ваш проект станет гибким, понятным и удобным для командной работы!
