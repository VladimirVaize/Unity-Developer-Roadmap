# 🎬 Анимация в Unity: Animator Controller и машина состояний

> [!Note]
> Этот материал посвящён системе анимации Unity, а именно Animator Controller, машине состояний (State Machine) и параметрам (Parameters).
> Эти инструменты позволяют создавать сложное, отзывчивое поведение персонажей и объектов — от простого переключения между "Idle" и "Run"
> до динамических переходов под влиянием скорости, здоровья или действий игрока.

---

## 🧠 Что такое Animator Controller?
Animator Controller — это графический ассет, который управляет анимациями объекта. Он содержит:
- Машину состояний (State Machine) — набор состояний и переходов между ними.
- Параметры (Parameters) — переменные, которые управляют переходами (скорость, здоровье, триггер прыжка и т.д.).

Чтобы использовать Animator Controller, прикрепите компонент `Animator` к игровому объекту и назначьте ему созданный контроллер.

---

## 🔁 Машина состояний (State Machine)
Машина состояний — это схема, где каждое состояние (State) — это один клип анимации (например, `Idle`, `Walk`, `Run`, `Jump`), а переходы (Transitions) определяют, когда и как состояние сменяется другим.

### Основные понятия:
- Any State — специальное состояние, от которого можно перейти в любое другое состояние из любого текущего состояния (удобно для событий вроде `Death` или `Jump`).
- Entry — точка входа в машину состояний (обычно ведёт в состояние по умолчанию, например `Idle`).
- Exit — выход из машины (используется редко, чаще для подсхем).

### Как создать и настроить:
1. Создайте Animator Controller: правой кнопкой в окне `Project` → `Create` → `Animator Controller`.
2. Откройте его двойным кликом — откроется окно `Animator`.
3. Перетащите анимационные клипы (например, `Idle`, `Walk`, `Run`) из окна `Project` в окно `Animator` — они станут состояниями (прямоугольники с названиями).
4. Установите состояние по умолчанию (оранжевый цвет): правой кнопкой по состоянию → `Set as Layer Default State`.
5. Создайте переход: правой кнопкой по состоянию → `Make Transition`, затем кликните на целевое состояние.

### Пример:
У вас есть `Idle` и `Walk`. Вы хотите, чтобы персонаж шёл, когда скорость больше 0.1. 
Создайте переход от `Idle` к `Walk` и настрой условие `Speed > 0.1`. 
Также создайте обратный переход от `Walk` к `Idle` с условием `Speed < 0.1`.

## 🎛️ Параметры (Parameters)
Параметры — это переменные, которые вы можете задавать из скриптов (через `Animator.SetFloat()`, `SetBool()`, `SetInteger()`, `SetTrigger()`). Они определяют условия переходов.

### Типы параметров:
| Тип | Описание | Пример использования |
|-------|----------------|-------------------------|
| Int | Целое число (0, 1, 2, -5) | Тип оружия (0 — меч, 1 — лук, 2 — посох) |
| Float | Число с плавающей точкой | Скорость движения, угол поворота |
| Bool | Истина или ложь (true/false) | IsGrounded (на земле), IsDead (мёртв) |
| Trigger | Как Bool, но автоматически сбрасывается в false после использования | Прыжок, удар, получение урона |

### Как создать параметр:
1. В окне `Animator` слева на вкладке `Parameters` нажмите `+`.
2. Выберите тип (`Float`, `Int`, `Bool`, `Trigger`).
3. Дайте имя (например, `Speed`, `IsJumping`, `Attack`).

---

## 🔀 Переходы (Transitions) и условия (Conditions)
Переход определяет: из какого состояния → в какое состояние. У перехода есть условия (Conditions) — это проверка параметров.

### Как настроить переход:
1. Кликните на стрелку перехода между двумя состояниями.
2. В инспекторе (Inspector) найдите раздел `Conditions`.
3. Нажмите `+`, выберите параметр и задайте значение.
   - Для `Float`: `Greater` / `Less` (например, `Speed > 0.1`)
   - Для `Int`: `Equals` / `Not Equal` (например, `WeaponType Equals 1`)
   - Для `Bool`: `True` / `False`
   - Для Trigger: просто наличие триггера (без значения)
  
### Дополнительные настройки перехода:
- Has Exit Time — если включено, переход произойдёт только после завершения анимации (длительность клипа). Для плавных переходов типа Idle→Run часто выключают.
- Transition Duration — время затухания между анимациями (по умолчанию ~0.1с для плавности).
- Conditions — список условий. Если их несколько, все должны быть истинны для перехода.

### Пример с Trigger:
1. Создайте параметр `Jump` типа `Trigger`.
2. В скрипте при прыжке вызовите `animator.SetTrigger("Jump")`.
3. Создайте переход от `Any State` к состоянию `Jump`.
4. В условии перехода выберите `Jump`.
5. Теперь в любой момент (Idle, Walk, Run) при вызове триггера сработает анимация прыжка.

## 🧩 Практический пример структуры
Представьте персонажа с анимациями: `Idle`, `Walk`, `Run`, `Jump`, `Attack`.

### Параметры:
- `Speed` (Float) — от 0 до 10 (скорость движения)
- `IsGrounded` (Bool) — стоит ли на земле
- `Attack` (Trigger) — удар
- `Jump` (Trigger) — прыжок

### Логика переходов:
- `Idle` → `Walk`: `Speed > 0.1`
- `Walk` → `Idle`: `Speed < 0.1`
- `Walk` → `Run`: `Speed > 5.0`
- `Run` → `Walk`: `Speed < 5.0`
- `Any State` → `Jump`: `Jump` (Trigger)
- `Any State` → `Attack`: `Attack` (Trigger)
- `Jump` → `Idle`: `IsGrounded == true` (после приземления)

---

## 💡 Советы
- Используйте Any State с осторожностью — он прерывает текущую анимацию мгновенно, что не всегда желательно (например, для прыжка хорошо, для атаки — лучше от определённых состояний).
- Для смешивания анимаций (например, поворот головы) используйте Layers и Avatar Masks, но это уже продвинутая тема.
- В скриптах обновляйте параметры в `Update()`: `animator.SetFloat("Speed", currentSpeed);`
