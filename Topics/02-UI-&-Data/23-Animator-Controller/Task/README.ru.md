# 🎯 Практическая задача: Создание Animator Controller для персонажа с бегом, прыжком и атакой

## Цель задачи
Научиться создавать Animator Controller, настраивать машину состояний и использовать параметры всех типов (Float, Bool, Trigger) для управления анимациями персонажа.

---

## Описание задачи
Вам нужно создать анимационный контроллер для простого персонажа с тремя базовыми действиями: покой (Idle), бег (Run), прыжок (Jump) и атака (Attack).

---

## Требования

### 1. Создание анимаций (клипов)
У вас есть следующие анимационные клипы (предположим, они уже импортированы в проект):
- `Idle` — персонаж стоит на месте
- `Run` — персонаж бежит
- `Jump` — персонаж подпрыгивает
- `Attack` — персонаж наносит удар

### 2. Параметры Animator Controller
Создайте в контроллере следующие параметры:

| Имя параметра | Тип | Назначение |
|------------|-------|----------------|
| `Speed` | Float | Скорость движения (от 0 до 10) |
| `IsGrounded` | Bool | Находится ли персонаж на земле |
| `Jump` | Trigger | Запуск прыжка |
| `Attack` | Trigger | Запуск атаки |

### 3. Логика переходов (настройте их в State Machine)

#### Базовые переходы:
- `Idle` → `Run`: когда `Speed > 0.1`
- `Run` → `Idle`: когда `Speed < 0.1`

#### Прыжок (из любого состояния):
- `Any State` → `Jump`: условие `Jump` (Trigger)
- `Jump` → `Idle`: условие `IsGrounded == true`

#### Атака (только из Idle и Run):
- `Idle` → `Attack`: условие `Attack` (Trigger)
- `Run` → `Attack`: условие `Attack` (Trigger)
- `Attack` → `Idle`: после завершения анимации атаки (включите `Has Exit Time`)

### 4. Дополнительные условия (по желанию, для усложнения)
- Если персонаж атакует в воздухе (`IsGrounded == false`), пусть атака не прерывает прыжок (сделайте переход от `Attack` только из `Idle` и `Run`, но не из `Jump`).
- Добавьте `ForceTransition` или используйте задержку атаки через `Transition Duration`.

---

## Что нужно сделать в Unity
1. Создайте Animator Controller с именем `PlayerAnimator`.
2. Добавьте в него состояния `Idle`, `Run`, `Jump`, `Attack` (перетащив клипы).
3. Создайте параметры (`Speed`, `IsGrounded`, `Jump`, `Attack`).
4. Создайте все переходы согласно логике выше.
5. Напишите простой C# скрипт, который:
   - Хранит ссылку на `Animator`.
   - В методе `Update()` получает ввод с клавиатуры:
     - `W, S, A, D` или `Horizontal` / `Vertical` — изменяет `Speed`.
     - Пробел (`Space`) — вызывает `SetTrigger("Jump")`.
     - Левый Ctrl или мышь — вызывает `SetTrigger("Attack")`.
   - Пример кода:
```csharp
void Update()
{
    float moveInput = Input.GetAxis("Vertical");
    float speed = Mathf.Abs(moveInput) * 10f;
    animator.SetFloat("Speed", speed);
    
    if (Input.GetButtonDown("Jump"))
        animator.SetTrigger("Jump");
    
    if (Input.GetKeyDown(KeyCode.LeftControl))
        animator.SetTrigger("Attack");
        
    // Для демо можно захардкодить IsGrounded = true всегда (если нет физики)
    animator.SetBool("IsGrounded", true);
}
```

6. Прикрепите `Animator` компонент к персонажу и назначьте созданный контроллер.
7. Запустите сцену и проверьте:
   - При нажатии `W/S` персонаж переключается между Idle и Run.
   - При пробеле — проигрывается Jump и возвращается в Idle.
   - При Ctrl — проигрывается Attack из Idle или Run и возвращается обратно.
  
---

## Критерии успеха
- ✅ Все состояния корректно переключаются.
- ✅ Параметры влияют на переходы.
- ✅ Триггеры `Jump` и `Attack` срабатывают при вызове из скрипта (и не повторяются постоянно).
- ✅ Переход `Jump` → `Idle` происходит только после `IsGrounded == true`.
