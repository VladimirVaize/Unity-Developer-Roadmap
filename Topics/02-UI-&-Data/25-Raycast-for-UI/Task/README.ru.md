# 🧩 Практическая задача: Блокировка стрельбы через UI

## 📝 Условие
### У вас есть простая сцена:
- 3D куб (мишень) с компонентом `Collider` и скриптом, который меняет цвет при клике.
- Кнопка `Fire` (UI) в углу экрана.
- Панель `Pause Panel` (UI), которая появляется по нажатию на клавишу `Space`. Панель занимает половину экрана и содержит текст «Пауза», а также кнопку `Resume`.

### Требования:
1. При клике на куб (через `Physics.Raycast`) он меняет цвет на случайный.
2. При клике на кнопку `Fire` в консоль выводится «Выстрел из UI», и куб не должен менять цвет.
3. Когда открыта панель паузы, клики по кубу не должны работать (куб не меняет цвет).
4. Панель паузы должна перехватывать клики, даже если курсор над «пустым» местом панели (не над кнопкой `Resume`).
5. При нажатии на кнопку `Resume` панель паузы закрывается.

---

## 🎯 Цель
Реализовать `Graphic Raycaster` и правильное использование `Raycast Target`, чтобы:
- Кнопка `Fire` никогда не активировала выстрел по кубу.
- Панель паузы блокировала все клики по кубу, пока она активна.

---

## 📋 Пошаговая подсказка
1. Добавьте на Canvas компонент `Graphic Raycaster` (если его нет).
2. Убедитесь, что в сцене есть `EventSystem`.
3. На кнопке `Fire` оставьте `Raycast Target` = true (по умолчанию).
4. На панели паузы:
   - Фоновая панель (Image) должна иметь `Raycast Target = true`.
   - Текст и кнопка `Resume` — стандартные настройки.
  
5. В скрипте для куба перед выполнением `Physics.Raycast` добавьте проверку:
```csharp
if (EventSystem.current.IsPointerOverGameObject())
    return;
```
6. При открытии паузы (`Space`) панель становится активной (`SetActive(true)`). При закрытии — неактивной.

---

## 🔍 Ожидаемый результат
- Клик точно по кубу при закрытой паузе → куб меняет цвет.
- Клик по кнопке `Fire` → в консоли сообщение, цвет куба не меняется.
- Открыта пауза → куб не реагирует на клики (даже если кликнуть прямо по нему).
- Клик по `Resume` → панель паузы исчезает, куб снова реагирует.
