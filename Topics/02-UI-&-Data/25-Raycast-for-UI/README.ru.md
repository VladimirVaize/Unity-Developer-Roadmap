# 🎯 Raycast для UI: Блокировка кликов и Graphic Raycaster

> [!Note]
> В Unity интерфейс пользователя (UI) работает по особым правилам.
> Обычный физический Raycast (лучевой пуск) не взаимодействует с UI-элементами.
> Для обработки кликов по кнопкам, панелям, изображениям и блокировки кликов сквозь
> UI используется система Graphic Raycaster и понятие блокировки кликов.

---

## 📦 Что такое Graphic Raycaster?
Graphic Raycaster — это компонент, который добавляется к Canvas (холсту) и отвечает за:
- Определение, на какой UI-элемент (Image, Button, Text и т.д.) нажал пользователь.
- Отправку событий (например, `IPointerClickHandler`) этим элементам.
- Блокировку физических лучей (или лучей от камеры), чтобы клик не прошёл «сквозь» UI в мир 3D/2D.

> Без Graphic Raycaster кнопки и другие UI-элементы не будут реагировать на клики.

---

## 🛡️ Блокировка кликов (как не дать кликнуть сквозь UI)
Когда поверх игрового мира висит окно инвентаря или паузы, клик должен поглощаться UI, а не стрелять в 3D-объекты.

### Как это работает:
1. При клике мыши Unity проверяет, есть ли под курсором UI-элемент.
2. Graphic Raycaster на Canvas обрабатывает все UI-объекты и определяет самый верхний.
3. Если UI-элемент найден, событие клика не передаётся физическому Raycast'у (например, от `Physics.Raycast`).
4. Таким образом, вы не сможете выстрелить сквозь панель инвентаря.

### Пример настройки:
- На Canvas уже есть компонент `Canvas Scaler` и `Graphic Raycaster`.
- На каждом UI-элементе (Button, Image) есть компонент `Image` с включённым `Raycast Target` (галочка).
  - Если снять эту галку, элемент станет «прозрачным» для лучей — клик пройдёт сквозь него дальше.

## 🧪 Для чего это нужно (примеры работы)

### 1. Блокировка взаимодействия с миром во время открытого UI
Ситуация: Игрок открывает инвентарь (`Canvas` с `Graphic Raycaster`). 
Пока инвентарь открыт, он не должен стрелять или подбирать предметы.

#### Как использовать:
- Просто положите UI-панель поверх всего экрана (например, полупрозрачный фон).
- Graphic Raycaster перехватит все клики.
- В скрипте проверяйте `EventSystem.current.IsPointerOverGameObject()` — если true, значит клик по UI, и вы игнорируете действия в мире.

### 2. Защита от сквозного нажатия кнопок
Ситуация: У вас есть большая кнопка «Старт». Если нажать чуть мимо неё, клик не должен уходить в игровой мир.

#### Как использовать:
- Установите на фоновую панель (или на саму кнопку) `Raycast Target = true`.
- Сделайте так, чтобы под кнопкой не было других интерактивных UI-элементов без необходимости.

### 3. Условная блокировка по слоям или типам элементов
Ситуация: Вы хотите, чтобы клик по пустому месту на UI закрывал окно, но проходил сквозь определённую декоративную иконку.

### Как использовать:
- На декоративной иконке снимите галочку `Raycast Target`.
- Тогда клик пройдёт сквозь неё и может быть пойман нижележащей панелью или даже 3D-миром.

## 🔧 Ключевые компоненты и настройки

| Компонент / Параметр | Назначение |
|----------------------------|-----------------------|
| `Graphic Raycaster` (на Canvas) | Включает UI-лучеметание. Без него UI не кликабелен. |
| `Raycast Target` (на UI-элементе) | Определяет, должен ли этот элемент перехватывать клики. |
| `EventSystem` (в сцене) | Обрабатывает ввод (мышь, касания). Без него лучи не работают. |
| `EventSystem.current.IsPointerOverGameObject()` | Метод в скрипте для проверки: нажат ли сейчас UI. |

---

## 💡 Частая ошибка
### Ошибка: 
- UI не реагирует на клики, хотя Canvas и кнопки есть.

### Причина: 
- На Canvas отсутствует компонент `Graphic Raycaster` или нет `EventSystem` в сцене.

### Решение:
- Выберите Canvas → `Add Component` → `Graphic Raycaster`.
- Если нет `EventSystem`, создайте: `GameObject` → `UI` → `Event System`.

---

## 🧠 Итог
- `Graphic Raycaster` — обязательный компонент для работы кликов по UI.
- Он автоматически блокирует прохождение кликов к 3D-объектам, если клик пришёлся по UI.
- Управлять «прозрачностью» отдельных элементов для лучей можно через флаг `Raycast Target`.
- Для проверки из скрипта используйте `EventSystem.current.IsPointerOverGameObject()`.
