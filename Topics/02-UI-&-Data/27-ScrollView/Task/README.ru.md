# 📝 Задача: Создать виртуализированный список чата с переиспользованием элементов

### Контекст:
Вы разрабатываете интерфейс игрового чата, который может содержать до 10 000 сообщений. 
Прямое создание 10 000 UI-объектов приведёт к падению производительности. 
Нужно реализовать динамический список с виртуализацией и переиспользованием элементов.

### Требования:
1. Создайте ScrollView (вертикальный) с `Viewport`, `Content` и вертикальным Scrollbar.
2. Реализуйте скрипт `VirtualChatList.cs`, который:
   - Хранит список сообщений (например, `List<string>`)
   - Создаёт пул из 10 UI-элементов (например, префаб `ChatMessagePrefab` с текстом `TextMeshProUGUI`)
   - При прокрутке вычисляет, какие индексы сообщений должны быть видны
   - Переиспользует элементы: берёт элемент из пула, который ушёл за пределы видимости, и заполняет его новым текстом
   - Подписывается на событие `OnValueChanged` ScrollRect
  
3. Добавьте тестовую генерацию данных:
   - Кнопка "Add 1000 messages" — добавляет 1000 сообщений в список данных и обновляет отображение
  
4. Индикатор производительности (опционально):
   - Отображайте количество физических объектов (должно быть всегда ~10) vs количество данных (растёт)
  
### Ожидаемый результат:
- Прокрутка списка из 10 000 сообщений плавная (60+ FPS)
- В иерархии всегда только 10–12 объектов (не 10 000)
- Новые сообщения подгружаются без задержек

### Подсказки:
- Используйте `RectTransform.anchoredPosition` для позиционирования элементов
- Высота каждого элемента фиксированная (например, 80 пикселей)
- При изменении размера списка или прокрутке вызывайте метод `Refresh()`
- Для буфера часто берут `visibleCount + 2` элемента
