# 📜 ScrollView и динамические списки в Unity UI

> [!Note]
> Этот материал посвящён созданию прокручиваемых списков, оптимизации производительности с помощью Virtual Layout и переиспользованию элементов.
> Эти техники критически важны для инвентарей, таблиц лидеров, чатов, магазинов и любых интерфейсов с большим количеством данных.

---

## 🧩 Основы ScrollView
ScrollView — это готовый UI-компонент Unity, который позволяет прокручивать содержимое, выходящее за пределы видимой области.

### Как создать ScrollView:
1. ПКМ в Hierarchy → `UI` → `Scroll View`
2. Unity автоматически создаст:
   - `ScrollView` (родительский объект с компонентом Scroll Rect)
   - `Viewport` (область, через которую видно содержимое)
   - `Content` (внутренний объект, который реально перемещается)
   - `Scrollbar Horizontal` и `Scrollbar Vertical` (полосы прокрутки)
  
### Настройка направления:
- Vertical — вертикальный список (галочка в компоненте Scroll Rect)
- Horizontal — горизонтальный список
- Оба — двумерная прокрутка

### Как добавлять элементы вручную:
Создайте элемент (например, кнопку или панель с текстом) как дочерний объект `Content`. 
При включенной вертикальной прокрутке новые элементы будут уходить вниз.

---

## ⚠️ Проблема больших списков (Naive подход)
Допустим, вы создаёте список из 1000 элементов, каждый с текстом, иконкой и кнопкой. 
Если вы создадите 1000 объектов UI как детей `Content`:
- Производительность резко упадёт — каждый объект требует CPU/GPU на рендеринг, обновление трансформаций, обработку событий.
- Время загрузки станет огромным.
- Потребление памяти вырастет многократно.

Это называется "наивное создание списка" (naive list) и не подходит для реальных проектов.

## 🚀 Virtual Layout (Виртуализация списка)
Virtual Layout — это техника, при которой создаются только видимые на экране элементы + небольшой буфер. 
Когда пользователь прокручивает список, элементы переиспользуются: те, что ушли за границы, заполняются новыми данными.

### Принцип работы:
1. Вы вычисляете, сколько элементов может поместиться на экране (например, 5).
2. Создаёте физически, допустим, 7 объектов (видимые + буфер).
3. При прокрутке определяете, какой индекс данных должен быть на позиции каждого объекта.
4. Обновляете текст, иконку, кнопку внутри существующего объекта (переиспользуете его).

### Преимущества:
- Огромная экономия памяти (вместо 1000 объектов — только 7)
- Мгновенная загрузка любого размера списка
- Плавная прокрутка

## ♻️ Переиспользование элементов (Recycling / Pooling)
Переиспользование — это сердце виртуализации. 
Вместо удаления и создания новых элементов при прокрутке вы берёте элемент, 
который ушёл за пределы видимости, меняете его данные и перемещаете в начало или конец списка.

### Простая модель переиспользования (с MLAPI, ListView и т.д.):
1. При старте создаётся пул из N элементов (например, 10).
2. На событии прокрутки вычисляется, какие индексы данных должны быть видны.
3. Для каждого видимого индекса:
   - Если уже есть элемент, которому назначен этот индекс — ничего не делаем.
   - Если нет — берём элемент из пула (или создаём новый, если пул ещё не заполнен).
   - Назначаем ему новый индекс и обновляем UI (загружаем текст, иконку и т.д.).
  
4. Элементы, которые больше не видны, возвращаются в пул.

### Пример на псевдокоде:
```csharp
void OnScroll()
{
    int firstVisibleIndex = CalculateFirstVisibleIndex();
    for (int i = 0; i < visibleCount; i++)
    {
        int dataIndex = firstVisibleIndex + i;
        GameObject item = GetRecycledItem(); // из пула
        item.GetComponent<MyItemView>().SetData(data[dataIndex]);
        item.transform.SetParent(content, false);
        PositionItem(item, i);
    }
}
```

## 🧰 Готовые решения в Unity

### 1. ScrollView + Content Size Fitter + Layout Group (для статических списков)
- Подходит: для небольших списков (< 50 элементов)
- Не подходит: для тысяч элементов

### 2. UI Toolkit (Unity 2021+)
- Встроенный ListView с виртуализацией
- `makeItem` и `bindItem` коллбэки

### 3. Ассеты из Asset Store
- `Super ScrollView`
- `EnhancedScroller`
- `UI Extensions` (Recycling ScrollRect)

### 4. Самодельный Virtual ScrollRect
Написать свой компонент, наследующий ScrollRect, и управлять позиционированием элементов через `RectTransform.anchoredPosition`.

---

## 📌 Итоговые рекомендации

| Размер списка | Подход |
|-------------------|-----------------------|
| < 30 элементов | Обычный ScrollView + Layout Group | 
| 30–200 элементов | Возможно обычный, но уже стоит задуматься |
| > 200 элементов | Обязательно Virtual Layout + переиспользование |
| > 1000 элементов | Виртуализация + асинхронная загрузка данных |

Золотое правило: Никогда не создавайте тысячи UI-объектов одновременно. Всегда переиспользуйте их.
