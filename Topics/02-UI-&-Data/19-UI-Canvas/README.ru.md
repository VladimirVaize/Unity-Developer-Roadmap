# 🎨 UI Canvas в Unity: Режимы рендеринга

**Canvas** — это компонент, на котором размещаются все элементы пользовательского интерфейса (UI): кнопки, тексты, панели, изображения. 
От того, в каком режиме рендеринга работает Canvas, зависит, как UI отображается на экране и взаимодействует с 3D-миром.

В Unity существует три режима рендеринга Canvas:

1. **Screen Space – Overlay** (Экранное пространство – поверх всего)
2. **Screen Space – Camera** (Экранное пространство – камера)
3. **World Space** (Мировое пространство)

---

## 🟢 1. Screen Space – Overlay (Поверх всего)

### Как работает:
Canvas накладывается прямо на экран, поверх всей 3D-сцены. 
UI всегда виден и не зависит от положения камеры. 
Элементы позиционируются в пиксельных координатах (например, x=100, y=200).

### Когда использовать:
- Главное меню, панели инвентаря, полоски здоровья (HUD)
- Диалоговые окна, которые всегда должны быть сверху
- Любой интерфейс, который не должен "уходить" за 3D-объекты

### Пример из работы:
Создайте Canvas (по умолчанию он уже в этом режиме). Добавьте кнопку по центру. 
Запустите сцену — кнопка будет всегда видна, даже если камера вращается или 3D-объекты её перекрывают.

### Как настроить:
В инспекторе Canvas → `Render Mode = Screen Space – Overlay`.

#### Дополнительно:
- `Pixel Perfect` — резкие пиксели (для ретро-игр)
- `Sort Order` — порядок отрисовки, если несколько Canvas

---

## 🔵 2. Screen Space – Camera (Экранное пространство – камера)

### Как работает:
Canvas "приклеен" к экрану, но рендерится через указанную камеру. 
Это позволяет UI взаимодействовать с 3D-миром — например, элементы могут частично перекрываться 3D-объектами, если те находятся между камерой и Canvas.

### Когда использовать:
- Эффект, когда 3D-модель проходит перед окном диалога
- 3D-тени или частицы на UI
- UI, который должен учитывать перспективу камеры (например, "голографические" экраны)

### Пример из работы:
1. Создайте Canvas, выберите `Render Mode = Screen Space – Camera`.
2. Перетащите в поле `Render Camera` вашу Main Camera.
3. Добавьте на сцену 3D-куб между камерой и Canvas (например, на расстоянии 2 единицы от камеры).
4. Запустите игру — куб будет перекрывать часть UI.

### Как настроить:
- `Plane Distance` — расстояние от камеры до плоскости Canvas. Чем меньше расстояние, тем ближе UI к камере.
- Если расстояние больше, чем до 3D-объекта, объект будет перекрывать UI.

---

## 🟣 3. World Space (Мировое пространство)

### Как работает:
Canvas становится 3D-объектом в сцене. Он имеет размер, поворот и положение. 
UI отображается как часть мира — например, как табличка над дверью, экран монитора в игре или всплывающие цифры урона над персонажем.

### Когда использовать:
- Интерфейс, привязанный к 3D-объекту (имя над головой NPC, полоска здоровья врага)
- Интерактивные экраны внутри игры (киоск, терминал)
- Диалоговые пузыри в 3D-пространстве

### Пример из работы:
1. Создайте Canvas, выберите `Render Mode = World Space`.
2. Задайте размер (например, `Width=3, Height=2`).
3. Добавьте на Canvas кнопку и текст.
4. Разместите Canvas как 3D-объект над дверью.
5. Запустите игру — теперь нужно подойти к двери, чтобы взаимодействовать с UI (например, Raycast из камеры).

### Как настроить:
- `Event Camera` — камера, которая будет обрабатывать клики по этому Canvas (обычно Main Camera).
- Масштабируйте Canvas через Transform, как любой 3D-объект.
- Для читаемости текста используйте `Dynamic Pixels Per Unit`.

## 📊 Сравнительная таблица (кратко)

| Режим | Где отображается | Зависимость от камеры | 3D-перекрытие | Типичное применение |
|-------------|---------------------|--------------------|-------------|-----------------------|
| Overlay | Поверх всего экрана | Нет | Нет | Меню, HUD, курсор |
| Camera | Привязан к экрану, но через камеру | Да | Частично | UI с 3D-эффектами |
| World Space | Как объект в сцене | Да (для Raycast) | Полное | Таблички, имена NPC, экраны |

---

## 🧠 Итог
- Overlay — самый простой и быстрый для статического UI.
- Camera — гибрид для эффектов глубины.
- World Space — для интерфейса, который живёт внутри игрового мира.

Выбор режима зависит от того, должен ли UI быть частью мира или всегда висеть перед глазами игрока.
