# 🧪 Практическая задача: Оптимизация стрельбы через пул объектов

## 🎯 Цель задачи
Научиться применять пул объектов для оптимизации производительности игры. Вы создадите систему стрельбы, где пули не создаются и не уничтожаются, а переиспользуются.

---

## 📝 Описание задачи
У вас есть простая игра-шутер с видом сверху:
- Игрок управляет персонажем (клик мыши или пробел для стрельбы).
- При каждом выстреле создаётся пуля (префаб `Bullet`), которая летит вперёд.
- Пуля уничтожается при попадании в стену или через 2 секунды.

### 🔴 Текущая проблема (плохая реализация):
```csharp
public class BadShooter : MonoBehaviour
{
    public GameObject bulletPrefab;
    
    void Update()
    {
        if (Input.GetMouseButtonDown(0))
        {
            // ❌ Аллокация каждый выстрел
            GameObject bullet = Instantiate(bulletPrefab, transform.position, Quaternion.identity);
            bullet.GetComponent<Rigidbody>().velocity = Vector3.forward * 20f;
            Destroy(bullet, 2f); // ❌ Вызовет работу GC
        }
    }
}
```
Проблема: 60 выстрелов → 60 аллокаций → 60 разрушений → сборщик мусора будет постоянно фризить игру.

---

## ✅ Задача
Перепишите систему стрельбы, используя пул объектов.

### Требования:
1. Создайте скрипт `BulletPool`, который:
   - Создаёт 30 пуль заранее (в `Awake` или `Start`).
   - Хранит их в `Queue<GameObject>` или использует `UnityEngine.Pool`.
   - Предоставляет метод `GetBullet()` (активирует пулю, выдаёт из пула).
   - Предоставляет метод `ReturnBullet(GameObject bullet)` (деактивирует, возвращает в пул).
  
2. Перепишите скрипт игрока `GoodShooter`:
   - Вместо `Instantiate` вызывает `bulletPool.GetBullet()`.
   - Не вызывает `Destroy` — вместо этого пуля сама возвращается в пул.
  
3. Напишите скрипт `Bullet`:
   - Двигает пулю вперёд (можно через `Rigidbody` или `Transform`).
   - При столкновении (`OnCollisionEnter`) вызывает `ReturnBullet`.
   - Возвращается в пул автоматически через 2 секунды (даже без столкновения).
  
4. Добавьте отладочный вывод (например, в `Update` пули), который показывает, активна ли пуля.

---

## 📤 Ожидаемый результат:
- В окне Hierarchy должно быть ровно 30 пуль (всегда).
- При выстреле одна из выключенных пуль включается и летит.
- После попадания или таймаута пуля выключается и возвращается в очередь.
- В Profiler → GC Alloc при стрельбе НЕ должно быть выделения памяти (0 байт).

## 🧠 Дополнительное задание (со звёздочкой):
- Добавьте автоматическое расширение пула: если пуль не хватает, создайте новую и добавьте в пул.
- Сделайте, чтобы пуля меняла цвет при активации и возвращала при деактивации.
- Используйте официальный `UnityEngine.Pool.ObjectPool<T>` вместо ручного `Queue`.
