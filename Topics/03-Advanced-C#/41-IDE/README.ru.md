# 🛠️ Настройка IDE и отладка в Unity

> [!Note]
> Этот материал посвящён тому, как настроить среду разработки (IDE) для работы с Unity,
> а также как использовать встроенные инструменты отладки: брейкпоинты, Immediate Window и Attach to Unity.
> Правильная отладка позволяет находить и исправлять ошибки в логике игры, не останавливая рабочий процесс.

---

## 1. Выбор и настройка IDE 🖥️
Unity официально поддерживает Visual Studio (Community/Professional) и Rider от JetBrains. 
Обе IDE имеют бесплатные версии для некоммерческого использования.

### Visual Studio

- **Установка**: При установке Unity Hub предложит установить Visual Studio с пакетом «Разработка игр на Unity».
  Также можно установить вручную через Visual Studio Installer, выбрав нагрузку *Game development with Unity*.

- **Настройка**: В Unity: `Edit → Preferences → External Tools → External Script Editor` → выберите `Visual Studio`.
  Убедитесь, что установлен пакет `Visual Studio Editor` в Package Manager.

### Rider

- **Установка**: Скачайте с официального сайта JetBrains.
  Rider имеет бесплатную лицензию для студентов и open-source проектов, а также 30-дневный триал.

- **Настройка**: В Unity: `Edit → Preferences → External Tools → External Script Editor` → выберите `Rider`.
  Rider автоматически импортирует Unity-проект и распознает все скрипты.

### 🔧 Проверка подключения
Откройте любой C# скрипт из Unity (двойной клик). Должна запуститься выбранная IDE. 
Если этого не происходит — проверьте путь в настройках External Tools.

## 2. Брейкпоинты (Breakpoints) 🛑
Для чего: Брейкпоинт — это маркер в коде, при достижении которого выполнение программы останавливается. 
Вы можете посмотреть значения переменных, состояние объектов и выполнять код пошагово.

### Как использовать:
- **Установка**: В IDE кликните на левую серую полосу напротив номера строки (появится красный кружок).
- **Снятие**: Кликните ещё раз.
- **Условные брейкпоинты**: ПКМ по красному кружку → `Conditions` → введите условие (например, `i == 5`).
  Остановка произойдёт только когда условие истинно.

### Пример:
```csharp
void Update()
{
    float speed = 10f;
    transform.Translate(Vector3.forward * speed * Time.deltaTime);
}
```
Поставьте брейкпоинт на строке `transform.Translate....` Запустите игру (Play в Unity). 
Когда объект начнёт движение, выполнение остановится. 
Теперь вы можете навести курсор на переменную `speed` и увидеть её значение.

## 3. Immediate Window (Окно непосредственного выполнения) ⌨️
Для чего: Immediate Window позволяет выполнять C# код во время остановки на брейкпоинте. 
Это мощный инструмент для проверки гипотез, изменения переменных или вызова методов на лету.

### Как использовать:
- Visual Studio: `Debug → Windows → Immediate` (или `Ctrl + Alt + I`)
- Rider: `View → Tool Windows → Immediate Window` (или ищите через `Ctrl + Shift + A`)

### Пример (продолжение):
Игра остановлена на брейкпоинте в `Update()`. В Immediate Window напишите:
```csharp
speed = 50;
```
Нажмите Enter. Теперь переменная `speed` изменилась прямо во время выполнения. Нажмите `Continue` (F5) — объект поедет быстрее, без перекомпиляции.

## 4. Attach to Unity (Подключение к Unity) 🔗
Для чего: Позволяет отлаживать код без запуска IDE из Unity. 
Вы уже запустили игру в редакторе, но забыли поставить брейкпоинты? Просто «прикрепитесь» к процессу Unity.

### Как использовать:
- **Visual Studio**: `Debug → Attach Unity Debugger` → выберите редактор Unity (обычно `Unity Editor (ваша_версия)`).
- **Rider**: Кнопка `Attach to Unity Process` (зелёный плей-треугольник с плюсом) или `Run → Attach to Unity Process`.
- **Автоматическое подключение**: В настройках IDE можно включить «Always attach» для текущего проекта.

> [!Important]
> На момент подключения Unity должна быть запущена (воспроизведение игры не обязательно). 
> После подключения вы можете поставить брейкпоинты и они сработают при следующем проходе кода.

### Пример:
Вы запустили игру, персонаж странно телепортируется. Вы не закрывая игру, в Rider нажимаете `Attach to Unity Process`. 
Открываете скрипт перемещения, ставите брейкпоинт в `Update()` — при следующем кадре выполнение остановится, и вы увидите, почему координаты меняются неправильно.

## 🔄 Типичный отладочный цикл
1. Пишете код в IDE.
2. Запускаете игру в Unity.
3. Если произошла ошибка или неожиданное поведение — ставите брейкпоинт.
4. Останавливаете игру, перезапускаете с прикреплённым отладчиком (или используете `Attach to Unity`).
5. Когда брейкпоинт срабатывает — изучаете переменные в Immediate Window.
6. Исправляете код, перекомпилируете (Unity перекомпилирует автоматически) и продолжаете.
