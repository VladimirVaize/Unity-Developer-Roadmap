# 🎯 Практическая задача: Система баффов и дебаффов через Reflection

## 📋 Описание задачи
Вам нужно создать гибкую систему баффов/дебаффов (временных эффектов), которая использует Reflection для изменения любых полей в любом скрипте, 
интерфейсы для применения эффектов, `[RequireComponent]` для автоматического добавления необходимых компонентов и `[ContextMenu]` для отладки.

---

## 🧱 Структура задачи
### 📁 Часть 1: Базовые классы и интерфейсы
1. Создайте интерфейс `IBuffable` с методом `ApplyBuff(BuffData data)`
2. Создайте класс `BuffData`, который содержит:
   - `string fieldName` — имя поля для изменения
   - `float valueModifier` — модификатор значения (например, 1.2 для увеличения на 20%)
   - `float duration` — длительность эффекта в секундах
  
### 📁 Часть 2: Скрипт с баффами
Создайте скрипт `BuffReceiver` с `[RequireComponent(typeof(IBuffable))]`. Он должен:
- Хранить активные баффы в списке/словаре
- Иметь метод `AddBuff(BuffData buff)`, который применяет изменение через Reflection
- Через определённое время восстанавливать исходное значение поля (используйте `Coroutine`)

### 📁 Часть 3: Тестовый класс
Создайте класс `PlayerStats : MonoBehaviour, IBuffable` с публичными полями:
- `float speed = 5.0f`
- `int damage = 10`
- `float jumpPower = 8.0f`

### 📁 Часть 4: Применение баффов
Создайте скрипт `BuffTester` с методом `[ContextMenu("Apply Speed Buff")]`, который:
- Находит на том же объекте компонент `IBuffable`
- Создаёт `BuffData` для изменения `speed` на 1.5 (увеличение скорости на 50%) на 5 секунд
- Применяет бафф

---

## ✅ Критерии выполнения
1. Используйте `System.Reflection` для чтения/изменения поля по имени (`GetType().GetField()`)
2. `GetComponents<IBuffable>()` должен находить все компоненты, реализующие интерфейс
3. `[RequireComponent(typeof(IBuffable))]` должен автоматически требовать наличие интерфейса
> (подсказка: интерфейс нельзя напрямую требовать через RequireComponent — подумайте, как обойти это ограничение, требуя конкретный компонент, который реализует интерфейс)

4. `[ContextMenu]` используется для быстрого тестирования баффов из редактора

---

## 🧩 Дополнительное задание (⭐)
Добавьте поддержку стекования баффов:
- Если на одно поле накладывается второй бафф, он заменяет предыдущий (сбрасывает таймер) ИЛИ складывается мультипликативно (×1.2, затем ×1.3 = ×1.56)

---

## 🧪 Ожидаемый результат
После добавления `BuffTester` на объект с `PlayerStats`:
- Клик правой кнопкой по компоненту `BuffTester` в Inspector → "Apply Speed Buff"
- Скорость игрока (`speed`) увеличивается с 5 до 7.5 на 5 секунд
- Через 5 секунд скорость возвращается к 5
- В консоли выводятся логи применения и окончания баффа
