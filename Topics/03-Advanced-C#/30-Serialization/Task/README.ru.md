# 🛠️ Практическая задача: «Инвентарь героя»

Вы создаёте RPG игру. Вам нужно реализовать систему инвентаря, где каждый предмет имеет название, вес и цену. 
Используйте знания о сериализации для правильного отображения и сокрытия полей в Инспекторе.

## 📝 Условие
1. Создайте структуру <a href="../Solution/InventoryItem.cs"><code>InventoryItem</code></a> с полями:
   - `itemName` (string, публичное) — название предмета.
   - `weight` (float, приватное, но должно отображаться в Инспекторе) — вес предмета.
   - `price` (int, публичное, но должно быть скрыто в Инспекторе) — цена предмета.
  
2. Создайте класс <a href="../Solution/Inventory.cs"><code>Inventory</code></a> (MonoBehaviour), который содержит:
   - Публичное поле `items` — массив из `InventoryItem`.
   - Приватное поле `maxWeight` (float), которое должно отображаться в Инспекторе.
   - Приватное поле `ownerName` (string), которое не должно отображаться в Инспекторе.
  
3. Напишите метод `GetTotalWeight()` внутри `Inventory`, который возвращает сумму весов всех предметов.
4. В методе `Start()` проверьте, не превышает ли общий вес `maxWeight`. Если превышает — выведите в консоль предупреждение: `"Инвентарь [ownerName] перегружен!"`.

---

## ✅ Что нужно использовать
- `[System.Serializable]` для структуры.
- `[SerializeField]` для приватных полей, которые должны быть видны в Инспекторе.
- `[HideInInspector]` для публичного поля, которое нужно скрыть.

---

## 🧪 Проверка результата
- Откройте Unity, создайте пустой GameObject, добавьте скрипт `Inventory`.
- В Инспекторе вы должны видеть:
  - Массив `Items`, и внутри каждого элемента — `Item Name` и `Weight` (но не `Price`).
  - Поле `Max Weight`.
  - Отсутствие поля `Owner Name`.
 
- Заполните несколько предметов, задайте `maxWeight`. Запустите сцену и посмотрите консоль.

> 💡 Подсказка: Чтобы скрыть `price` в структуре, используйте `[HideInInspector] public int price;`.
> Несмотря на сокрытие, значение сохранится и будет доступно из кода.
