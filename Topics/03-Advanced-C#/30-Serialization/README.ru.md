# 📦 Сериализация в Unity: Управление видимостью полей в Инспекторе

Сериализация — это процесс преобразования данных объектов в формат, который может быть сохранён (например, в сцену или префаб) и восстановлен позже. 
Unity автоматически сериализует публичные поля (public) большинства типов, отображая их в окне Инспектор. Однако этого часто недостаточно. 
В этом материале мы разберём три ключевых атрибута для точной настройки сериализации: `[SerializeField]`, `[HideInInspector]` и `[System.Serializable]`.

## 🎯 Зачем управлять сериализацией?
- Скрыть внутренние переменные, которые не должны редактироваться в инспекторе.
- Показать в инспекторе `private` или `protected` поля без нарушения инкапсуляции.
- Работать со сложными типами данных (структуры, классы), которые Unity не сериализует по умолчанию.

---

## 1. `[SerializeField]` — Показать приватное поле
### Для чего нужно:
Unity по умолчанию не сериализует приватные (`private`, `protected`, `internal`) поля. 
Атрибут `[SerializeField]` заставляет Unity показать такое поле в Инспекторе и сохранить его значение, не делая поле публичным.
Это помогает соблюдать инкапсуляцию.

### Как использовать:
- Добавьте `[SerializeField]` перед объявлением приватного поля.
- Поле появится в Инспекторе, но из других скриптов к нему нельзя будет обратиться напрямую (без специальных методов).

### 📌 Пример:
```csharp
using UnityEngine;

public class PlayerHealth : MonoBehaviour
{
    [SerializeField] private int maxHealth = 100;  // Видно в Инспекторе, но приватно
    [SerializeField] private float invincibilityDuration = 1.5f;

    private int currentHealth;

    void Start()
    {
        currentHealth = maxHealth;
    }
}
```

> В Инспекторе вы увидите поля `Max Health` и `Invincibility Duration`, сможете их менять.
> При этом другой скрипт не сможет случайно изменить `maxHealth` напрямую.

## 2. `[HideInInspector]` — Скрыть публичное поле
### Для чего нужно:
Обратная ситуация: у вас есть публичное поле (public), которое Unity показывает в 
Инспекторе автоматически, но вы не хотите, чтобы дизайнер или вы сами его там видели и меняли. 
Например, техническая переменная, ссылка, которая устанавливается кодом.

### Как использовать:
- Добавьте `[HideInInspector]` перед `public` полем.
- Поле останется публичным (доступным из других скриптов), но исчезнет из Инспектора.

### 📌 Пример:
```csharp
using UnityEngine;

public class EnemyAI : MonoBehaviour
{
    [HideInInspector] public Transform playerTarget; // Назначается кодом, не нужно редактировать вручную
    public float attackRange = 5f;                   // Видно и редактируется

    void Start()
    {
        playerTarget = GameObject.FindGameObjectWithTag("Player").transform;
    }
}
```

> `playerTarget` не засоряет Инспектор, хотя другие скрипты могут его читать.

## 3. `[System.Serializable]` — Сериализация своих классов и структур
### Для чего нужно:
Unity не умеет автоматически сериализовать обычные классы и структуры, определённые вами. 
Если вы объявите `public MyCustomClass data` в MonoBehaviour, в Инспекторе вы ничего не увидите. 
Атрибут `[System.Serializable]` говорит Unity: «Этот тип можно сериализовать — покажи его поля в Инспекторе, сохраняй в сцене/префабе».

### Как использовать:
- Добавьте `[System.Serializable]` перед определением класса или структуры.
- Внутри используйте публичные поля или `[SerializeField]` приватные поля.
- Затем объявите переменную этого типа в вашем скрипте.

### 📌 Пример со структурой:
```csharp
using UnityEngine;

[System.Serializable]
public struct WeaponStats
{
    public string weaponName;
    public int damage;
    public float fireRate;
}

public class PlayerCombat : MonoBehaviour
{
    public WeaponStats mainWeapon;   // Появится в Инспекторе как сворачиваемый блок
    [SerializeField] private WeaponStats secondaryWeapon; // Тоже видно, но приватно
}
```

### 📌 Пример с классом:
```csharp
[System.Serializable]
public class InventorySlot
{
    public string itemID;
    public int quantity;
    [HideInInspector] public int internalIndex; // Поле внутри сериализуемого класса тоже можно скрыть
}

public class Inventory : MonoBehaviour
{
    public InventorySlot[] slots; // В Инспекторе вы увидите массив с редактируемыми элементами
}
```

> [!Important]
> Для классов Unity требует, чтобы они были не абстрактными, имели конструктор без параметров и поля были сериализуемых типов.

## 📊 Сводная таблица

| Атрибут | Применение к | Эффект в Инспекторе | Доступ из кода |
|--------------|--------------|------------------|------------------|
| `[SerializeField]` | `private` / `protected` | ✅ Показывает | ❌ Прямого нет |
| `[HideInInspector]` | `public` | ❌ Скрывает | ✅ Есть (public) |
| `[System.Serializable]` | Свои классы/структуры | ✅ Позволяет видеть поля | Зависит от уровня доступа полей |

---

## ⚠️ Частые ошибки
1. Забыть `[System.Serializable]` → в Инспекторе вместо полей будет пустая надпись или ничего.
2. Использовать `[SerializeField]` на свойствах (properties с `get/set`) → Unity их не сериализует.
3. Пытаться сериализовать `Dictionary` или `Queue` → они не поддерживаются из коробки (нужны обходные пути).
4. Скрыть поле `[HideInInspector]`, но ожидать, что оно загрузит сохранённое значение → оно сохранится, но вы не сможете его увидеть в редакторе (что иногда полезно).
