# ⚙️ OnValidate: Автоматическая настройка пресетов в Unity Editor

`OnValidate` — это специальный метод в Unity, который автоматически вызывается только в редакторе (не в сборках игры). 
Он срабатывает всякий раз, когда скрипт загружается или когда вы изменяете значения его полей в окне Inspector. 
Это идеальный инструмент для автоматической проверки, корректировки и настройки параметров объекта прямо во время разработки.

---

## 🎯 Для чего нужен `OnValidate`?
- ✅ Автоматическая коррекция значений — например, не давать радиусу стать отрицательным.
- ✅ Обновление дочерних объектов — изменение цвета материала сразу на всех частях префаба.
- ✅ Валидация пресетов — убедиться, что настройки объекта соответствуют правилам (например, `MaxHealth` > 0).
- ✅ Визуальная отладка — обновление отображения в Scene View при изменении параметров.
- ✅ Настройка пресетов (Presets) — при применении пресета `OnValidate` может докалибровать другие поля.

> [!Important]
> `OnValidate` не вызывается в рантайме (в сборках игры). Он работает ТОЛЬКО в редакторе Unity.

---

## 🧠 Синтаксис и использование
Добавьте метод в любой класс, наследующий от `MonoBehaviour`:
```csharp
using UnityEngine;

public class Weapon : MonoBehaviour
{
    [SerializeField] private int damage = 10;
    [SerializeField] private float range = 5f;

    private void OnValidate()
    {
        // Автоматическая подстройка значений
        if (damage < 1)
            damage = 1;

        if (range < 0.5f)
            range = 0.5f;

        Debug.Log($"[OnValidate] Оружие настроено: урон {damage}, радиус {range}");
    }
}
```

Теперь, когда вы поменяете `damage` в инспекторе на 0 — `OnValidate` сразу же вернёт его к 1.

## 🔧 Примеры работы
### 1. Защита от дурака (Clamping)
```csharp
private void OnValidate()
{
    health = Mathf.Max(1, health);
    speed = Mathf.Clamp(speed, 0f, 20f);
}
```

### 2. Авто-обновление дочерних объектов
```csharp
private void OnValidate()
{
    var renderers = GetComponentsInChildren<Renderer>();
    foreach (var rend in renderers)
        rend.sharedMaterial.color = teamColor;
}
```

### 3. Синхронизация двух полей
```csharp
private void OnValidate()
{
    if (maxHealth < currentHealth)
        currentHealth = maxHealth;
}
```

### 4. Работа с пресетами (Presets)
Если пользователь применит пресет к вашему компоненту, `OnValidate` автоматически подстроит зависимые поля.
```csharp
private void OnValidate()
{
    attackSpeed = Mathf.Max(0.2f, attackSpeed);
    attackCooldown = 1f / attackSpeed; // автообновление
}
```

---

## 🧩 Когда НЕ использовать `OnValidate`
- ❌ В сборках игры (бесполезно — не вызовется).
- ❌ Для тяжёлых операций (создание/удаление объектов, поиск по сцене) — может тормозить редактор.
- ❌ Для логики, которая должна работать в рантайме — используйте `Awake`, `Start`, `Update`.
