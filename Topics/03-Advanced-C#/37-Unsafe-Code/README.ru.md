# 🧠 Unsafe код и Native Containers: Когда и как использовать

Этот материал посвящён работе с небезопасным (unsafe) кодом и Native контейнерами (в частности `NativeArray`) в Unity. 
Эти инструменты необходимы для высокопроизводительных вычислений, особенно при использовании Job System и Burst Compiler. 
Мы разберём, что это такое, когда их применение оправдано, а когда — нет.

---

## 🚨 Что такое Unsafe код?

В C# (и Unity) "безопасный" код работает под управлением сборщика мусора (Garbage Collector, GC) и проверяет границы массивов. Unsafe код позволяет:
- Использовать указатели (как в C/C++)
- Работать с напрямую выделенной памятью (stackalloc, Marshal.AllocHGlobal)
- Отключать проверки границ массивов для скорости

### Как включить:
`Project Settings → Player → Allow 'unsafe' Code` (галочка). В скрипте добавьте unsafe к классу или методу.
```csharp
unsafe void FastCopy(float* source, float* dest, int count) 
{
    for (int i = 0; i < count; i++) dest[i] = source[i];
}
```
> [!Warning]
> Unsafe код не проверяется GC и может привести к утечкам памяти, повреждению данных и крашам. Используйте только при острой необходимости!

## 📦 NativeArray и другие Native Containers
Native контейнеры — это структуры, которые хранят данные в "нативной" памяти (не в управляемой куче GC). 
Они нужны для передачи данных в Jobs, поскольку Jobs не могут обращаться к управляемым объектам.

### Основные типы:
| Контейнер            | Назначение                                          |
| ---                  | ---                                                 |
| `NativeArray<T>`     | Упорядоченный список элементов (как обычный массив) |
| `NativeList<T>`      | Динамический список                                 |
| `NativeHashMap<K,V>` | Словарь (хэш-таблица)                               |
| `NativeQueue<T>`     | Очередь                                             |

### Создание NativeArray:
```csharp
NativeArray<float> data = new NativeArray<float>(1000, Allocator.Temp);
// Allocator.Temp — быстрый, до 4 кадров
// Allocator.Persistent — до ручного освобождения
// Allocator.TempJob — для коротких Jobs (4 кадра)

// Всегда освобождайте!
data.Dispose();
```

### Использование в Job:
```csharp
struct MyJob : IJob
{
    public NativeArray<float> results;
    public void Execute()
    {
        for (int i = 0; i < results.Length; i++)
            results[i] = results[i] * 2;
    }
}
```

## ⚖️ Когда использование Unsafe и NativeArray оправдано?
### ✅ Оправдано (оптимизация производительности):
- Обработка тысяч/миллионов объектов (частицы, лучи, сетка вершин)
- Real-time физика/анимация (должна работать >60 FPS)
- Использование Job System + Burst (NativeArray обязателен)
- Чтение больших бинарных файлов (свои форматы уровней/моделей)
- Низкоуровневые алгоритмы (A* поиск пути, конвекция тепла, симуляция жидкости)

### ❌ НЕ оправдано (усложнение без выгоды):
- Простые сценарии с <100 объектами
- UI-логика, ввод, работа с сетью (там узкое место не в CPU)
- Командные проекты без опытного разработчика (риск трудноуловимых багов)
- Прототипирование (быстрота разработки важнее 20% прироста скорости)

### 💡 Золотое правило:
> Сначала пишите безопасный managed-код.
> Если профилировщик (Profiler) показывает, что конкретный участок — узкое место, тогда рефакторите его под unsafe/NativeArray.

## 🔍 Пример: Сравнение скорости
```csharp
// Медленный способ (GC + проверки границ)
void ManagedProcess(float[] array)
{
    for (int i = 0; i < array.Length; i++)
        array[i] = Mathf.Sin(array[i]);
}

// Быстрый способ (Unsafe + NativeArray)
unsafe void UnsafeProcess(NativeArray<float> array)
{
    float* ptr = (float*)array.GetUnsafePtr();
    for (int i = 0; i < array.Length; i++)
        ptr[i] = Mathf.Sin(ptr[i]);
}
```
Разница при 1 млн элементов: ~10-20 мс (managed) против ~1-3 мс (unsafe + Burst). Но для 100 элементов разница незаметна.

---

## 🧹 Важно: Управление памятью
Все Native контейнеры нужно вручную удалять через `.Dispose()`, иначе утечка памяти (memory leak). 
Для защиты используйте `using` или `try-finally`:
```csharp
NativeArray<int> arr = new NativeArray<int>(100, Allocator.Temp);
try
{
    // работа с arr
}
finally
{
    arr.Dispose();
}
```
Или автоматический `Dispose` с помощью `DisposeHandle` в JobSystem.

## 🧪 Итог
| Характеристика   | Безопасный код | Unsafe + NativeArray               |
| ---              | ---            | ---                                |
| Скорость         | 🐢 Средняя    | 🚀 Очень высокая                   |

| Проверка границ  | ✅ Да         | ❌ Нет (можно вылететь за пределы) |
| Сложность        | 📗 Низкая     | 📕 Высокая                         |
| Где использовать | Почти везде    | Только хотспоты (5-10% кода)       |
