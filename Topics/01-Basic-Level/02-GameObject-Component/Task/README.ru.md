# 🎯 Практическое задание: «Собери робота из компонентов»

## Цель

Научиться различать GameObject (контейнер) и Component (поведение/часть). Создать работающий объект, добавляя и удаляя компоненты.

## Задание

### Часть 1. Создайте пустой контейнер

1. Откройте Unity (любая версия 2020+)
2. В окне Hierarchy → ПКМ → `Create Empty`
3. Назовите объект `MyRobot`
4. Посмотрите в Inspector: какие компоненты уже есть? (только `Transform`)

### Часть 2. Добавьте визуальное поведение (компоненты)

1. Выделите `MyRobot`
2. Нажмите `Add Component` → `Mesh Filter` → выберите `Cube` (или `Sphere`)
3. Добавьте `Mesh Renderer` (если не добавился автоматически)
4. Создайте материал: ПКМ в Project → `Create` → `Material`, назовите `RobotMat`, цвет — красный
5. Перетащите `RobotMat` на `MyRobot` (или назначьте в `Mesh Renderer` → `Element 0`)

➡️ Теперь `MyRobot` видим. Но он не двигается и не взаимодействует.

### Часть 3. Добавьте физическое поведение

1. `Add Component` → `Rigidbody`
2. `Add Component` → `Box Collider` (автоматически подстроится под размер куба)

➡️ Нажмите Play. Объект упадёт вниз под гравитацией. Остановите игру.

### Часть 4. Добавьте логическое поведение (скрипт)

1. Создайте C# скрипт: ПКМ в Project → `Create` → `C# Script`, назовите `Rotator`
2. Откройте скрипт (двойной клик) и напишите:

```csharp
using UnityEngine;

public class Rotator : MonoBehaviour
{
    public float speed = 30f;

    void Update()
    {
        transform.Rotate(Vector3.up, speed * Time.deltaTime);
    }
}
```

3. Сохраните скрипт. Перетащите его на `MyRobot` в Inspector (или на объект в Hierarchy)

### Часть 5. Эксперименты (проверка понимания)

Выполните по очереди и наблюдайте результат:

1. Отключите `Mesh Renderer` (снимите галочку) → объект стал невидимым, но вращается (если Play) — потому что `Rotator` всё ещё работает.
2. Отключите `Rotator` → вращение остановилось.
3. Удалите `Rigidbody` (три точки → Remove Component) → гравитация и физика пропали.
4. Создайте второй объект (`Create Empty` → назовите `MyRobot2`) и перетащите на него готовый скрипт `Rotator` из Project. Он тоже будет вращаться.

### Часть 6. Ответьте на вопросы (устно или в комментарии к коду)

- Сколько GameObject в сцене после части 5? (подсказка: каждый объект в Hierarchy — это GameObject)
- Можно ли добавить два одинаковых компонента `Rigidbody` на один GameObject? (проверьте)
- Если удалить `Transform` — что произойдёт?

> ✅ Критерий успеха: Вы можете создать объект, сделать его видимым, физическим, с логикой вращения — и объяснить, какая часть является контейнером (GameObject), а какие — компонентами.
