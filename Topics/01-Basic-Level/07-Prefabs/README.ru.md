# 📦 Префабы (Prefabs) в Unity: Создание, изменение, вложенные префабы


> [!NOTE]
>Префабы — это один из самых мощных инструментов Unity. Они позволяют создавать многократно используемые шаблоны игровых объектов.
>Вместо того чтобы 10 раз настраивать одного и того же врага, вы делаете его один раз как префаб, а затем просто «штампуете» экземпляры в сцене.
>Когда вы меняете оригинал префаба — все его копии обновляются автоматически.

---

## 🔧 1. Создание префаба (Creating a Prefab)

### Как сделать префаб:

1. В Иерархии (Hierarchy) создайте и настройте любой игровой объект: добавьте компоненты (Rigidbody, скрипт), задайте спрайт/модель, цвета, параметры.
2. Перетащите этот объект из окна Иерархия в окно Проект (Project) (обычно в папку `Assets/Prefabs`).
3. Объект в иерархии станет синим — это значит, что теперь это экземпляр префаба. Исходный префаб (синяя иконка) хранится в окне «Проект».

### Пример: 🧠
Вы создали «Умного врага» со скриптом `EnemyAI`, здоровьем 100 и звуком атаки. Перетащили его в `Assets/Prefabs` → готово. 
Теперь перетаскивайте его в любую сцену 20 раз, и каждый враг будет вести себя одинаково.

## ✏️ 2. Применение изменений (Applying Changes)

Есть два способа изменить все экземпляры префаба:

### Способ A — Редактирование исходного префаба (прямое)
- Дважды кликните на префабе в окне Проект. Откроется режим редактирования префаба (Prefab Mode) — чистое окно только с этим объектом.
- Внесите изменения (например, увеличьте здоровье с 100 до 150).
- Нажмите `Save` (кнопка в левом верхнем углу) или просто закройте Prefab Mode. Все экземпляры в сцене обновятся.

### Способ B — Через экземпляр в сцене (Overrides)
1. Выберите любой экземпляр префаба в Иерархии.
2. Измените что-то в Инспекторе (например, позицию, цвет материала).
3. В Инспекторе появятся кнопки `Overrides` (выделяется синим).
4. Нажмите `Overrides` → выберите:
   - `Apply All` — применить все изменения к исходному префабу (все экземпляры изменятся).
   - `Revert All` — отменить изменения только на этом экземпляре.
   - `Apply Selected` — применить только конкретное изменение.
  
### Пример: 🎨
Вы поставили 5 копий дерева-префаба. Одному экземпляру в сцене вы изменили масштаб, сделав его больше. Другим — нет. 
Затем вы нажали `Apply` на этом изменении масштаба → теперь все деревья станут большими. 
Если нажать `Revert` — это дерево вернётся к исходному маленькому размеру.

## 🧩 3. Вложенные префабы (Nested Prefabs)

Это когда один префаб содержит внутри другой префаб. 
Например, префаб «Комната» содержит префабы «Стул», «Стол», «Лампа». 
И каждый из них можно изменять независимо.

### Как создать вложенный префаб:
1. Создайте префаб `Стул` (перетащите в Project).
2. Создайте префаб `Комната` (например, пустой GameObject).
3. Перетащите экземпляр `Стул` из Project прямо на `Комнат`у в Иерархии (как дочерний объект).
4. Теперь `Комната` — родительский префаб, внутри которого лежит дочерний префаб `Стул`.

### Что это даёт:
- Если вы измените исходный префаб `Стул` (цвет, размер) — все стулья во всех комнатах обновятся автоматически (и внутри `Комнаты`, и отдельно).
- Если вы измените префаб `Комната` (например, добавите окно) — это не затронет отдельные стулья.
- Можно редактировать вложенный префаб прямо внутри родительского: дважды кликните по `Комната` →
- внутри Prefab Mode вы увидите дочерний `Стул` (он будет синим). Кликните по нему → Open — откроется редактирование самого `Стула`.

### Пример сложного использования: 🏰
Префаб `Рыцарь` содержит префаб `Меч`. Если вы улучшите `Меч` (увеличите урон с 10 до 20) — все рыцари во всех сценах начнут бить сильнее. 
Но при этом у каждого рыцаря может быть свой цвет плаща (это изменение уже не влияет на всех, если не применить через Overrides).

## ⚠️ Важные нюансы

| Действие | Результат |
|----------------------|-----------------------|
| Изменение свойства у экземпляра (позиция, цвет) | Меняется ТОЛЬКО этот экземпляр (Override) |
| `Apply` этого изменения | Меняется исходный префаб → все экземпляры |
| Удаление компонента у экземпляра | Компонент пропадает только у этого экземпляра (Override) |
| `Revert All` на экземпляре | Все его оверрайды сбрасываются, он становится точной копией оригинала |
| Разрыв связи (`Unpack Prefab` в контекстном меню) | Превращает экземпляр в обычный GameObject без связи с префабом |

---

## 🎯 Когда использовать префабы?

- ✅ Одинаковые враги, пули, деревья, монеты, здания.
- ✅ UI элементы (кнопки, панели, окна инвентаря).
- ✅ Целые уровни или комнаты (как вложенные префабы).
- ✅ Любой объект, который повторяется в проекте более 1-2 раз.
