# 🧱 События столкновений в Unity: OnCollisionEnter и OnTriggerEnter

> [!Important]
> В Unity физические взаимодействия между объектами обрабатываются через два основных типа событий:
> Collision (реальное столкновение с отталкиванием) и Trigger (триггер, без физического отклика).
> Они вызываются автоматически, если на объектах присутствуют компоненты `Collider` и (опционально) `Rigidbody`.

---

## 🎯 Основные отличия

| Характеристика | OnCollisionEnter | OnTriggerEnter |
|----------------|------------------------------|------------------------------|
| Физический отклик | ✅ Есть (объекты отталкиваются) | ❌ Нет (объекты проходят сквозь) |
| Необходим Rigidbody | Хотя бы у одного из объектов | Хотя бы у одного из объектов (обычно у движущегося) |
| Параметр метода	 | `Collision collision` | `Collider other` |
| Режим коллайдера | `Collider` (без IsTrigger) | `Collider` с галочкой IsTrigger = true |
| Типичное применение | Столкновения шаров, машин, падающих предметов | Зоны урона, сбор предметов, триггерные двери, чекпоинты |

---

## ⚙️ Порядок вызова событий
Для двух объектов A и B, которые врезаются друг в друга:

### 1. OnCollisionEnter / OnTriggerEnter — вызывается один раз в момент начала контакта.
### 2. OnCollisionStay / OnTriggerStay — вызывается каждый кадр, пока контакт продолжается.
### 3. OnCollisionExit / OnTriggerExit — вызывается один раз, когда объекты перестают касаться.

> [!Note]
> 💡 Если оба объекта имеют `Rigidbody` и оба могут двигаться, события вызываются на обоих объектах.

---

## 🚫 Ограничения и важные правила

### Для OnCollisionEnter:
- Оба объекта должны иметь Collider (не Trigger).
- Хотя бы один из объектов должен иметь некинематический `Rigidbody` (т.е. `isKinematic = false`).
- Если оба объекта — кинематические (`isKinematic = true`), событие не возникнет.
- Событие не вызывается, если объекты просто пересекаются по триггерам.

### Для OnTriggerEnter:
- Хотя бы у одного из объектов `Collider` должен иметь **IsTrigger = true**.
- Хотя бы у одного из объектов должен быть `Rigidbody` (кинематический или нет — неважно).
- Если оба коллайдера — триггеры, событие всё равно вызывается (если есть Rigidbody).
- Триггер не генерирует сил, но вы можете вручную менять позиции, здоровье, удалять объекты и т.д.

---

## 📝 Примеры кода

```csharp
// Пример OnCollisionEnter (наносим урон при столкновении)
void OnCollisionEnter(Collision collision)
{
    if (collision.gameObject.CompareTag("Enemy"))
    {
        health -= 10;
        Debug.Log("Меня ударил враг!");
    }
}

// Пример OnTriggerEnter (подбираем предмет)
void OnTriggerEnter(Collider other)
{
    if (other.CompareTag("Coin"))
    {
        Destroy(other.gameObject);
        score += 100;
    }
}
```

---

## 🔁 Резюме
- **Collision** → физика + отскок + точное столкновение.
- **Trigger** → обнаружение входа в зону без физики.
- Для работы обязателен `Rigidbody` на одном из участников.
- Не забывайте проверять теги, чтобы не реагировать на все подряд объекты.
