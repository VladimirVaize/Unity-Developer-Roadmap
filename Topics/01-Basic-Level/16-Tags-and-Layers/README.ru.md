# 🏷️ Теги (Tag) и Слои (Layer) в Unity

> [!Note]
> Теги и слои — это мощные инструменты для организации, фильтрации и управления взаимодействием объектов в Unity.
> Они позволяют эффективно решать задачи от поиска объектов до сложной настройки физических столкновений.

---

## 🧷 Что такое Теги (Tags)?

### Для чего нужно:
Тег — это простая текстовая метка, которую можно присвоить игровому объекту. Несколько объектов могут иметь один и тот же тег. 
Теги используются для логической группировки объектов (например, «Player», «Enemy», «Collectable») и для их поиска без необходимости хранить прямые ссылки.

### Как использовать:
1. Выберите объект в Иерархии (Hierarchy).
2. В окне Инспектор (Inspector) найдите выпадающий список Tag (в самом верху, под именем объекта).
3. Выберите существующий тег (например, `Player`, `Finish`, `EditorOnly`) или создайте новый:
   - Нажмите `Add Tag...` → откроется менеджер тегов.
   - Введите имя нового тега и нажмите `Save`.
   - Теперь ваш новый тег доступен для выбора.
4. В коде (C#) вы можете проверять тег объекта или находить объекты по тегу:
```csharp
// Проверка тега
if (otherGameObject.CompareTag("Player"))
{
    // Делаем что-то с игроком
}

// Поиск объекта по тегу
GameObject player = GameObject.FindWithTag("Player");

// Поиск всех объектов с тегом
GameObject[] enemies = GameObject.FindGameObjectsWithTag("Enemy");
```

### Пример использования:
В игре есть монеты (тег `"Coin"`). При касании игрока с монетой вы проверяете: 
`if (other.CompareTag("Coin")) { Destroy(other.gameObject); Score += 10; }`. 
Так монеты не нужно специально регистрировать в отдельных списках.

## 🗂️ Что такое Слои (Layers)?

### Для чего нужно:
Слои — это числовые категории (от 0 до 31), которые позволяют управлять видимостью камер, 
игнорировать определённые объекты при лучепуске (Raycast) и настраивать физические взаимодействия между объектами. 
В отличие от тегов, слои обычно используют для массового исключения или фильтрации.

### Как использовать:
1. Выберите объект → в Инспекторе найдите выпадающий список Layer.
2. Выберите существующий слой (`Default`, `TransparentFX`, `Ignore Raycast`, `Water`, `UI`) или создайте новый:
   - Нажмите `Add Layer...` → откроется окно настроек (Tag Manager).
   - В полях `User Layer 8` ... `User Layer 31` введите имена своих слоёв (например, `Player`, `Enemy`, `Environment`, `Projectile`).
   - Вернитесь к объекту и назначьте ему нужный слой.
3. Настройка взаимодействий физики (Physics Matrix):
   - `Edit` → `Project Settings` → `Physics` (или `Physics 2D` для 2D).
   - Найдите секцию Layer Collision Matrix.
   - Это таблица, где строки и столбцы — это слои. Отметьте или снимите галочки, чтобы разрешить/запретить столкновения между определёнными слоями.
   > Пример: Вы не хотите, чтобы пули (слой `Projectile`) сталкивались с другими пулями, но должны сталкиваться с врагами (слой `Enemy`).
   > В матрице уберите галочку на пересечении `Projectile × Projectile`, оставьте галочку на `Projectile × Enemy`.
4. Использование маски слоёв в Raycast:<br>
   Маска слоя (LayerMask) позволяет лучу (Raycast) игнорировать все объекты, кроме выбранных слоёв.
```csharp
// Создаём маску, включающую только слои "Player" и "Enemy"
int layerMask = (1 << LayerMask.NameToLayer("Player")) | (1 << LayerMask.NameToLayer("Enemy"));

// Или: всё, кроме слоя "Ignore Raycast"
layerMask = ~(1 << LayerMask.NameToLayer("Ignore Raycast"));

// Используем в Raycast
RaycastHit hit;
if (Physics.Raycast(transform.position, transform.forward, out hit, 100f, layerMask))
{
    Debug.Log("Попали в: " + hit.collider.gameObject.name);
}
```

### Пример использования:
Камера следит за игроком. Вы бросаете луч из камеры в сторону игрока, чтобы проверить, не загораживает ли что-то обзор (стена). Стенам назначен слой `Environment`. 
Маска луча исключает слой `Player` (чтобы луч проходил сквозь игрока) и включает все остальные. Если луч попал в `Environment`, значит обзор заблокирован.

## ⚙️ Physics Matrix (Матрица взаимодействий)

### Для чего нужно:
Определяет, какие пары слоёв могут физически сталкиваться (вызывать события `OnCollisionEnter`, `OnTriggerEnter`) и влиять на движение (Rigidbody).

### Как настраивать:
  - Перейдите в `Project Settings` → `Physics` (или `Physics 2D`).
  - Найдите `Layer Collision Matrix`.
  - По умолчанию все слои могут сталкиваться со всеми.
    Снимите галочку на пересечении двух слоёв — объекты этих слоёв будут проходить друг сквозь друга без физического отклика
    (но триггеры всё равно могут срабатывать, если включены).

### Практический пример:
  - Слой `Player` и слой `Enemy` — сталкиваются (галочка есть).
  - Слой `Player` и слой `Collectable` — сталкиваются (галочка есть), но Collectable — триггер.
  - Слой `Projectile` и слой `Projectile` — НЕ сталкиваются (галочка снята), чтобы пули не сталкивались друг с другом.
  - Слой `Projectile` и слой `Player` — сталкиваются (галочка есть), чтобы игрок получал урон.

## 🆚 Теги (Tag) vs Слои (Layer) – когда что использовать?

| Задача | Тег (Tag) | Слой (Layer) |
|-----------------------------------|----------|----------|
| Поиск конкретного объекта (например, игрока) | ✅ `FindWithTag` | ❌ Не предназначен |
| Групповая проверка в `OnTriggerEnter` | ✅ `CompareTag` | ✅ `gameObject.layer` |
| Игнорирование целой группы при Raycast | ❌ Неэффективно | ✅ LayerMask |
| Настройка физических столкновений (кто с кем сталкивается) | ❌ Невозможно | ✅ Physics Matrix |
| Фильтрация отрисовки камеры (Culling Mask) | ❌ | ✅ |

### Лучшая практика:
  - Используйте теги для логической идентификации (кто это? – игрок, враг, бонус).
  - Используйте слои для системных настроек (как физика, видимость, лучи).
