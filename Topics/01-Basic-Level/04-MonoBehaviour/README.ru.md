# 🎬 MonoBehaviour: Жизненный цикл скрипта

Каждый скрипт в Unity, унаследованный от `MonoBehaviour`, проходит определённый порядок вызовов методов. Этот порядок называется жизненным циклом. 
Понимание того, когда какой метод вызывается, критически важно для правильной работы физики, ввода, анимации и оптимизации.

---

## ⏱️ Порядок вызовов (основные методы)

Unity вызывает эти методы автоматически в строгой последовательности:

```
Пробуждение → Активация → Старт → Повторяющиеся кадры → Фиксированные обновления → Поздние обновления → Отключение / Уничтожение
```

### 1. `Awake()` — Пробуждение 🟢
- Вызывается один раз, когда объект загружается в сцену (даже если скрипт выключен).
- Используется для инициализации ссылок между компонентами (GetComponent, поиск других объектов).
- Гарантированно вызывается до `Start()`.

### 2. `OnEnable()` — Включение 🟡
- Вызывается каждый раз, когда объект становится активным (вкл/выкл в иерархии или скрипте).
- Хорошее место для подписки на события.

### 3. `Start()` — Старт 🔵
- Вызывается один раз перед первым `Update()`, но только если скрипт включен.
- Используется для логики, зависящей от других объектов, которые уже инициализированы в `Awake()`.

### 4. `Update()` — Кадровое обновление 🟠
- Вызывается каждый кадр (частота зависит от FPS: 30, 60, 144+ раз в секунду).
- Используется для: обработки ввода (`Input.GetKey`), движения без физики, анимации, таймеров (Time.deltaTime).
- Не подходит для работы с физикой!

### 5. `FixedUpdate()` — Фиксированное обновление 🔴
- Вызывается с постоянным интервалом времени (по умолчанию 0.02 сек = 50 раз в секунду), не зависит от FPS.
- Используется для всего, что связано с физикой: `Rigidbody.AddForce`, перемещение через `MovePosition`, изменение скорости.
- Интервал можно изменить в `Project Settings → Time → Fixed Timestep`.

### 6. `LateUpdate()` — Позднее обновление 🟣
- Вызывается каждый кадр сразу после `Update()`.
- Используется для: следования камеры за игроком (чтобы камера двигалась после того, как игрок уже обновил позицию), процедурной анимации.

### 7. `OnDisable()` — Выключение ⚫
- Вызывается каждый раз, когда объект становится неактивным.
- Отписка от событий, остановка корутин.

### 8. `OnDestroy()` — Уничтожение ⚪
- Вызывается один раз при удалении объекта (Destroy) или закрытии сцены/приложения.
- Финальная очистка (если нужна).

---

## 📊 Визуальная схема (один объект, скрипт включен):

```
Загрузка сцены:
  Awake() → OnEnable() → Start()

Далее каждый кадр:
  FixedUpdate() → [физика] → Update() → LateUpdate()
  (FixedUpdate может вызываться несколько раз между Update, если FPS низкий)

При выключении объекта:
  OnDisable()

При уничтожении/выходе:
  OnDestroy()
```

---

## 🔧 Примеры использования

| Метод | Когда использовать | Что НЕ делать |
|--------|------------------------------|------------------------------|
| `Awake` | Найти компонент: <br>`GetComponent<Rigidbody>()` | Обращаться к другим объектам, которые могут ещё не проснуться |
| `Start` | Установить начальное здоровье, включить ИИ | Зависить от того, что `Awake` другого объекта мог не выполниться |
| `Update` | Движение: <br>`transform.Translate(Input.GetAxis("Horizontal") * speed * Time.deltaTime)` | Применять силы к Rigidbody (это в FixedUpdate) |
| `FixedUpdate` | Прыжок: <br>`rigidbody.AddForce(Vector3.up * jumpForce, ForceMode.Impulse)` | Использовать `Time.deltaTime` (вместо него `Time.fixedDeltaTime`) |
| `LateUpdate` | Камера следит за игроком: <br>`cameraTransform.position = playerTransform.position + offset` | Менять физическое положение Rigidbody |

---

## ⚠️ Важные замечания

- Если скрипт выключен (галочка снята в инспекторе), `Start()`, `Update()`, `FixedUpdate()`, `LateUpdate()` не вызываются, но `Awake()` и `OnEnable()` — вызываются (если объект активен).
- Если объект выключен в иерархии — ничего из жизненного цикла не вызывается.
- `Time.deltaTime` — время между кадрами (для `Update`).
- `Time.fixedDeltaTime` — фиксированный интервал (для `FixedUpdate`).
