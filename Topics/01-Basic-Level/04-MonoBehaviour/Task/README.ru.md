# 🧪 Практическая задача: Жизненный цикл MonoBehaviour

## 🎯 Цель
Создать скрипт, который демонстрирует порядок вызова методов жизненного цикла, и правильно распределить логику движения, физики и камеры между `Update()`, `FixedUpdate()` и `LateUpdate()`.

---

## 🧱 Условие

### У вас есть:
1. 🟦 Куб (Player) с компонентом Rigidbody.
2. 🎥 Камера (Camera), которая не является дочерней по отношению к кубу.
3. ⌨️ Управление: стрелки / WASD для движения куба по плоскости (X и Z).
4. ⬆️ Пробел — прыжок.

---

## ✅ Задачи

### 1. Создайте скрипт `PlayerMovement` и повесьте его на куб.
### 2. Реализуйте правильную структуру:
  - `Awake()` — получить компонент `Rigidbody` и сохранить ссылку.
  - `Start()` — установить начальную позицию куба в `(0, 1, 0)`.
  - `Update()` — считывать ввод (горизонталь/вертикаль и пробел).
  - `FixedUpdate()` — применять движение через `rigidbody.MovePosition()` (или `AddForce`) и прыжок через `AddForce`.
  - `LateUpdate()` — заставить камеру следовать за кубом с отставанием (плавное следование).
### 3. Добавьте `Debug.Log` в каждый метод, чтобы видеть порядок вызовов в консоли.

---

## 🔍 Ожидаемое поведение

- Куб движется плавно и без проваливания сквозь пол (если у пола есть `Collider`).
- Прыжок работает только когда куб на земле (используйте `OnCollisionEnter` или дополнительную проверку).
- Камера двигается после куба, не дрожит.
- В консоли видна последовательность: `Awake` → `Start` → (цикл `FixedUpdate` → `Update` → `LateUpdate`).
