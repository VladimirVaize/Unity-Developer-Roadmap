# 🧱 Коллайдеры (Colliders), Триггеры (Triggers) и Физические материалы (Physic Materials)

> [!NOTE]
> Этот материал объясняет, как работает физическое взаимодействие объектов в Unity.
> Вы узнаете разницу между обычным столкновением (Collision) и триггером (Trigger),
> а также как настроить трение и отскок с помощью физических материалов.

---

## 📦 Что такое коллайдер (Collider)?

> Коллайдер — это невидимый компонент, который добавляется к игровому объекту, чтобы определить его форму для физического движка. Без коллайдера объекты просто проходят друг сквозь друга.

### Основные типы коллайдеров:
- `Box Collider` — куб / параллелепипед.
- `Sphere Collider` — сфера.
- `Capsule Collider` — капсула (для персонажей).
- `Mesh Collider` — точная форма по сетке модели (дорого по производительности).

### Как добавить:
Выделите объект → `Add Component` → `Box Collider` (или другой).

> [!WARNING]
> Для движения с физикой (силы, скорость, гравитация) объекту также нужен компонент `Rigidbody`.

---

## ⚔️ Collision vs Trigger — в чём разница?

> У каждого коллайдера есть флажок Is Trigger. От него зависит тип взаимодействия.

| Режим | `Is Trigger` = false (по умолчанию) | `Is Trigger` = true |
|----------|------------------------------------|-----------------------|
| Поведение | Физическое столкновение — объекты отталкиваются, не могут проходить друг сквозь друга | Детектор — объекты проходят сквозь, но вы получаете событие входа |
| События в коде | `OnCollisionEnter`, `OnCollisionStay`, `OnCollisionExit` | `OnTriggerEnter`, `OnTriggerStay`, `OnTriggerExit` |
| Пример | Мяч отскакивает от стены | Дверь открывается, когда игрок входит в невидимую зону |

---

## 🔨 Как использовать:

### Обычное столкновение (Collision):
1. Добавьте коллайдер обоим объектам.
2. Добавьте `Rigidbody` хотя бы одному (лучше движущемуся).
3. Оставьте `Is Trigger = false`.
4. Напишите скрипт с `OnCollisionEnter`.

```csharp
void OnCollisionEnter(Collision collision)
{
    Debug.Log("Удар с: " + collision.gameObject.name);
}
```

### Триггер (Trigger):
1. Добавьте коллайдер.
2. Поставьте галочку `Is Trigger = true`.
3. Добавьте `igidbody` (можно `is Kinematic = true`, если объект не должен двигаться физически).
4. Используйте `OnTriggerEnter`.

```csharp
void OnTriggerEnter(Collider other)
{
    Debug.Log("Вошёл в зону: " + other.gameObject.name);
}
```

> [!IMPORTANT]
>🎯 Правило: Для триггера достаточно, чтобы один из двух объектов имел Rigidbody (обычно игрок или активный объект). Для коллизии нужны Rigidbody + коллайдеры у обоих.

## 🧴 Физические материалы (Physic Material)
> Физический материал определяет, как поверхности взаимодействуют при трении и отскоке. Без него объекты скользят бесконечно или ведут себя неестественно.

### Свойства:

| Параметр | Что делает |
|--------------|-------------------------------|
| `Dynamic Friction` | Трение при движении (0 = лёд, 1 = резина) |
| `Static Friction` | Трение в состоянии покоя (чтобы объект не начинал скользить сам) |
| `Bounciness` | Эластичность отскока (0 = нет отскока, 1 = идеальный отскок) |
| `Friction Combine` | Как смешивать трение двух объектов (Average, Minimum, Maximum, Multiply) |
| `Bounce Combine` | Как смешивать отскок двух объектов |

### Как создать и применить:
1. В окне Project → правой кнопкой → `Create` → `Physic Material`.
2. Назовите его, например, `Rubber` или `Ice`.
3. Настройте параметры (`Bounciness = 0.8`, `Dynamic Friction = 0.1`).
4. Перетащите материал на поле Material в компоненте коллайдера любого объекта.

### Примеры материалов:
- 🧊 Лёд: `Dynamic Friction = 0.05`, `Bounciness = 0.1`
- 🏀 Мяч: `Dynamic Friction = 0.4`, `Bounciness = 0.8`
- 🧱 Кирпич: `Dynamic Friction = 0.9`, `Bounciness = 0.1`

## 🧠 Итоговая таблица решений

| Что нужно | Что использовать |
|--------------|--------------------------|
| Мяч отскакивает от пола | Collision + Physic Material с Bounciness |
| Игрок собирает монету | Trigger + `OnTriggerEnter` |
| Машина тормозит на асфальте | Collision + Physic Material с высоким трением |
| Невидимая зона, открывающая дверь | Trigger |
| Пуля отскакивает от стены | Collision + Physic Material с Bounciness |
| Пол скользкий как лёд | Collision + Physic Material с трением ~0 |
